Detailed Description

USB request block–USB data transfer entity.

Parameters
lengthThe length of data that will be sent or received.
actualThe actual length of data that has been sent or received.
bufThe buffer that stores data will be sent or has been received.
zlpSend a zero-length packet when the length is equal to the maximum packet size of the endpoint.
complete_in_isr1: The complete callback will be called in ISR.
0: The complete callback will not be called in ISR, and the URB will be transferred through HAL_USB_COMMON_ISR_XFER_DONE.
statusThe status of the data transfer.
ep_handleThe handle of endpoint that will transfer data.
completeThe callback will be called when data that has been sent or received.
For sending data, this callback is mainly used to indicate the result of data sending.
For receiving data, this callback is used to get the data already received and related information.
privThe private data.
