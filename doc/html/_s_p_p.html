SPP Serial Port Profile

1 Introduction

The purpose of this document is to give an overview of the Serial Port Profile(SPP). SPP is used to send data through Bluetooth between two devices.

1.1 SPP Role Featrues

SPP roles are typically defined as Figure 1-1.

  • Device A (DevA) - This is the device that takes initiative to form a connection to another device.
    • SPP client
  • Device B (DevB) - This is the device that waits for another device to take initiative to connect.
    • SPP server
      Figure 1-1 SPP Proifle Scenario


1.2 SPP Profile Stack

Protocols and entities used in SPP profile shown in Figure 1-2.

  • The Baseband LMP and L2CAP are the OSI layer 1 and 2 Bluetooth protocols.
  • RFCOMM is the Bluetooth adaptation of GSM TS 07.10 , providing a transport protocol for serial port emulation.
  • SDP is the Bluetooth Service Discovery Protocol.
    Figure 1-2 Protocol Model


1.3 SPP UUID

  • Universally Unique Identifier(UUID) is a universally unique identifier that is guaranteed to be unique across all space and all time. UUIDs can be independently created in a distributed fashion. No central registry of assigned UUIDs is required. A UUID is a 128-bit value.
  • The full 128-bit value of a 16-bit or 32-bit UUID may be computed by a simple arithmetic operation:
    • 128_bit_value = 16_bit_value * 296 + Bluetooth_Base_UUID
    • 128_bit_value = 32_bit_value * 296 + Bluetooth_Base_UUID
    • The Base UUID(Bluetooth_Base_UUID) is used for calculating 128-bit UUIDs from 'short UUIDs' (uuid16 and uuid32) as described in the SDP Specification: 00000000-0000-1000-8000-00805F9B34FB
    • A 16-bit UUID may be converted to 32-bit UUID format by zero-extending the 16-bit value to 32-bits. An equivalent method is to add the 16-bit UUID value toa zero-valued 32-bit UUID.
  • Blutetooth SDP UUID numbers is defined in https://www.bluetooth.com/specifications/assigned-numbers/service-discovery/.
  • SPP UUID in SPP server and SPP client should be the same.
  • Different SPP applications shall use different SPP UUIDs.

2 SPP API Introduction

This manual describes the SPP API definition.

2.1 SPP Init

bool bt_spp_init()
Initialize SPP profile, and configure the maximum link number and supported service number.

bool bt_spp_service_register()
Register SPP service using SPP service uuid and local server channel.

2.2 SPP Connect

bool bt_spp_connect_req()
Send an SPP connection request.

bool bt_spp_connect_cfm()
Send an SPP connection confirmation.

bool bt_spp_disconnect_req()
Send an SPP disconnection request.

bool bt_spp_disconnect_all_req()
Send a request to disconnect all SPP connection.

2.3 SPP Command

bool bt_spp_registered_uuid_check()
Check the uuid is registered and bond local server channel with the checked uuid.

bool bt_spp_ertm_mode_set()
Set SPP ERTM mode.

bool bt_spp_data_send()
Send SPP data to remote device.

bool bt_spp_credits_give()
Give SPP credits to remote device.

3 Source code overview

3.1 Initialization

Initialize SPP, configure the maximum link number and supported service number. Register SPP service using SPP service uuid and local server channel. Register SPP porfile callback handler with bt_mgr_cback_register.

void app_spp_init(void)
{
if (app_cfg_const.supported_profile_mask & SPP_PROFILE_MASK)
{
uint8_t service_num = 1;
if (app_cfg_const.supported_profile_mask & SPP_PROFILE_MASK)
{
service_num++;
}
bt_spp_init(app_cfg_const.spp_link_number, service_num);
bt_spp_service_register((uint8_t *)spp_service_class_uuid128, RFC_SPP_CHANN_NUM);
bt_mgr_cback_register(app_spp_bt_cback);
}
}

  • For Serial Port Profile, 16-bit UUID 0x1101 UUID_SERIAL_PORT is reserved, meanwhile, for Vendor Serial Port, 128-bit vendor UUID may be defined and used by the user.

3.3 SPP Profile Callback Handler

  • app_spp_bt_cback function is used to handle BT Manager SPP related events.
  • Different SPP service may register different SPP profile callback handler by bt_mgr_cback_register.
  • SPP client:
    • SPP profile connection is completed when receives BT_EVENT_SPP_CONN_CMPL. SPP link credit means the SPP tx transport window size and SPP frame size means the SPP tx/rx transport MTU size.
    • If Spp link credit is zero, then, it will be set according to T_BT_EVENT_PARAM_SPP_CREDIT_RCVD when recieve BT_EVENT_SPP_CREDIT_RCVD. AnyWay, if SPP link credits is not zero, SPP client could send the tx data.
    • Spp client recieve the SPP data from SPP server when receives BT_EVENT_SPP_DATA_IND. meanwhile, it should call bt_spp_credits_give to update the SPP server tx credit(SPP client rx window size).
  • SPP server:
    • SPP profile connection requst should be confirmed when receives BT_EVENT_SPP_CONN_IND.
    • SPP profile connection is completed when receives BT_EVENT_SPP_CONN_CMPL. SPP link credit means the SPP tx transport window size and SPP frame size means the SPP tx/rx transport MTU size.
    • If Spp link credit is zero, then, it will be set according to T_BT_EVENT_PARAM_SPP_CREDIT_RCVD when recieve BT_EVENT_SPP_CREDIT_RCVD. AnyWay, if SPP link credits is not zero, SPP server could send the tx data.
    • Spp server recieve the SPP data from SPP client when receives BT_EVENT_SPP_DATA_IND. meanwhile, it should call bt_spp_credits_give to update the SPP client tx credit(SPP server rx window size).
      static void app_spp_bt_cback(T_BT_EVENT event_type, void *event_buf, uint16_t buf_len)
      {
      T_BT_EVENT_PARAM *param = event_buf;
      T_APP_BR_LINK *p_link;
      bool handle = true;
      switch (event_type)
      {
      case BT_EVENT_SPP_CONN_CMPL:
      {
      T_APP_BR_LINK *p_link;
      p_link = app_find_br_link(param->spp_conn_cmpl.bd_addr);
      if (p_link != NULL)
      {
      p_link->rfc_credit = param->spp_conn_cmpl.link_credit;
      p_link->rfc_frame_size = param->spp_conn_cmpl.frame_size;
      }
      }
      break;
      case BT_EVENT_SPP_CREDIT_RCVD:
      {
      p_link = app_find_br_link(param->spp_credit_rcvd.bd_addr);
      if (p_link == NULL)
      {
      APP_PRINT_ERROR0("app_spp_bt_cback: no acl link found");
      return;
      }
      if ((p_link->rfc_credit == 0) && (param->spp_credit_rcvd.link_credit))
      {
      app_pop_data_transfer_queue(CMD_PATH_SPP, false);
      }
      p_link->rfc_credit = param->spp_credit_rcvd.link_credit;
      }
      break;
      case BT_EVENT_SPP_DATA_IND:
      {
      uint8_t *p_data;
      uint16_t len;
      uint8_t app_idx;
      uint16_t data_len;
      uint16_t total_len;
      p_link = app_find_br_link(param->spp_data_ind.bd_addr);
      if (p_link == NULL)
      {
      APP_PRINT_ERROR0("app_spp_bt_cback: no acl link found");
      return;
      }
      app_idx = p_link->id;
      p_data = param->spp_data_ind.data;
      len = param->spp_data_ind.len;
      data_len = len;
      bt_spp_credits_give(app_db.br_link[app_idx].bd_addr, param->spp_data_ind.local_server_chann, 1);
      }
      break;
      case BT_EVENT_SPP_CONN_IND:
      {
      p_link = app_find_br_link(param->spp_conn_ind.bd_addr);
      if (p_link == NULL)
      {
      APP_PRINT_ERROR0("app_spp_bt_cback: no acl link found");
      return;
      }
      uint8_t local_server_chann = param->spp_conn_ind.local_server_chann;
      uint16_t frame_size = param->spp_conn_ind.frame_size;
      bt_spp_connect_cfm(p_link->bd_addr, local_server_chann, true, frame_size, 7);
      }
      break;
      case BT_EVENT_SPP_DISCONN_CMPL:
      {
      p_link = app_find_br_link(param->spp_disconn_cmpl.bd_addr);
      if (p_link != NULL)
      {
      p_link->rfc_credit = 0;
      }
      app_transfer_queue_reset(CMD_PATH_SPP);
      }
      break;
      default:
      handle = false;
      break;
      }
      if (handle == true)
      {
      APP_PRINT_INFO1("app_spp_bt_cback: event_type 0x%04x", event_type);
      }
      }
