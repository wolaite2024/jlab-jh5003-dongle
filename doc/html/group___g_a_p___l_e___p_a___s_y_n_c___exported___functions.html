T_GAP_CAUSE le_pa_sync_init ( uint8_t  sync_handle_num)

Initialize the number of sync handles for synchronization state in periodic advertising.

Note
If PA synchronization state will be used, le_ext_scan_gap_msg_info_way (false) should be invoked.
Parameters
[in]sync_handle_numSync handle number.
Returns
GAP Operation result.
Return values
GAP_CAUSE_SUCCESSGAP operation success.
GAP_CAUSE_INVALID_PARAMOperation failure, invalid parameter.
GAP_CAUSE_NO_RESOURCEOperation failure, memory acquisition failure.

Example usage

void test(void)
{
T_GAP_CAUSE cause = le_pa_sync_init(sync_handle_num);
}

◆ le_pa_sync_modify_periodic_adv_list()

T_GAP_CAUSE le_pa_sync_modify_periodic_adv_list ( T_GAP_PA_SYNC_PERIODIC_ADV_LIST_OP  operation,
uint8_t *  adv_addr,
T_GAP_PA_SYNC_ADV_ADDR_TYPE  adv_addr_type,
uint8_t  adv_sid 
)

Modify Periodic Advertiser list.

Note
This function to add, remove or clear Periodic Advertiser list can be called after Bluetooth Host is ready.
Explanation: If Bluetooth Host is ready, the application will be notified by message GAP_MSG_LE_DEV_STATE_CHANGE with new_state about gap_init_state which is configured as GAP_INIT_STATE_STACK_READY.

If sending request operation is success, the result of modifying Periodic Advertiser list will be returned by app_gap_callback with cb_type GAP_MSG_LE_PA_SYNC_MODIFY_PERIODIC_ADV_LIST.

Parameters
[in]operationAdd entry to Periodic Advertiser list, remove entry from Periodic Advertiser list or clear all entries from Periodic Advertiser list. T_GAP_PA_SYNC_PERIODIC_ADV_LIST_OP.
If operation is GAP_PA_SYNC_PERIODIC_ADV_LIST_OP_CLEAR, adv_addr, adv_addr_type and adv_sid are irrelevant.
[in]adv_addrPublic Device Address, Random Device Address, Public Identity Address, or Random (static) Identity Address of the advertiser.
[in]adv_addr_typeOnly PA_SYNC_ADV_ADDR_PUBLIC and PA_SYNC_ADV_ADDR_RANDOM could be used for modifying Periodic Advertiser list.
[in]adv_sidAdvertising SID subfield in the ADI field used to identify the Periodic Advertising.
Returns
Send request operation.
Return values
GAP_CAUSE_SUCCESSSend request operation success.
OthersSend request operation failure.

Example usage

void test(void)
{
T_GAP_CAUSE cause = le_pa_sync_modify_periodic_adv_list(operation, adv_addr, adv_addr_type, adv_sid);

T_GAP_CAUSE le_pa_sync_create_sync ( T_GAP_PA_SYNC_CREATE_SYNC_PARAM *  p_pa_sync_create_sync_param,
uint8_t *  p_sync_id 
)

Synchronize with a periodic advertising train from an advertiser.

This function can be called after Bluetooth Host is ready.
Explanation: If Bluetooth Host is ready, the application will be notified by message GAP_MSG_LE_DEV_STATE_CHANGE with new_state about gap_init_state which is configured as GAP_INIT_STATE_STACK_READY.

If sending request operation is success, the result of synchronization creation will be returned by app_gap_callback with cb_type GAP_MSG_LE_PA_SYNC_DEV_STATE_CHANGE_INFO, GAP PA synchronization states will be returned by app_gap_callback with cb_type GAP_MSG_LE_PA_SYNC_STATE_CHANGE_INFO. Periodic advertisement will be returned by app_gap_callback with cb_type GAP_MSG_LE_PERIODIC_ADV_REPORT_INFO.

Parameters
[in]p_pa_sync_create_sync_paramGAP PA Sync create sync parameter. T_GAP_PA_SYNC_CREATE_SYNC_PARAM.
[in,out]p_sync_idIdentify the periodic advertising train.
Returns
Send request operation.
Return values
GAP_CAUSE_SUCCESSSend request operation success.
OthersSend request operation failure.
