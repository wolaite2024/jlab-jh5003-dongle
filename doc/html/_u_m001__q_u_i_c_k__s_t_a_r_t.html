1 Overview

This document is aimed to help beginners to use Realtek MCU Software Development Kit (SDK) quickly for the first time and to make sure users can compile and generate images, set up EVB and software enviroment, run and debug applications on EVB. This document also guides users to view other user manuals.

The following topics are included:

  • The software and hardware needed during the development
  • EVB and software environment setup
  • How to run applications on EVB
  • Simple introduction of debugging methods
  • List the documents that need to view

2 Prerequisites

The following software and hardware will be needed during the development:

  • Keil MDK-ARM Essential V5 (or higher)
  • Git Bash
  • J-Link Software v6.32 or later
  • Installation Package
  • EVB Kit

2.1 Keil

All applications in the SDK could be compiled and used with Keil Microcontroller Development Kit(MDK). Before starting software development, Keil should be obtained and installed correctly by the users themselves. Please visit www.keil.com for more information.

Tool chain version as below is used by Realtek. This version or higher is suggested to be used to avoid any compatiblility issues between the BT stack in ROM and user applications.

2.2 Git Bash

As bash.exe will be used in the pre-build or after-build command line, please install Git from https://git-scm.com/downloads . After Git is installed, take the installation in 'C:\Program Files' as an example, and confirm that the environment variable path is added with 'C:\Program Files\Git\usr\bin'.

2.3 J-Link

J-Link is another prerequisite if a more comprehensive debugging method is desired besides the logging mechanism. To install the SEGGER J=Link software, visit www.segger.com to download related software and documentation. The driver should be correctly installed for the device to use the J-Link debugger with Keil MDK.

2.4 Installation Package

All materials of the SDK installation package would be installed into users' PC, including sample projects, documentation and essential tools. The SDK directory structure would be introduced in 5.1 SDK Directory in SDK User Guide , and users could develop their own application after setting the path correctly.

2.5 EVB Kit

The EVB with required hardware for evaluating, developing and debugging user applications on BT Audio is designed.

3.2.2 Project Introduction

RWS project is designed based on substantial BT and audio application experience, and has been fully verified. All the audio functions could be achieved by SDK. The developer could use this RWS project for customized design. The dedicated MCU configuration tool is provided to set different behavior without any modification in this RWS project.

The project is briefly introduced below if source code modification is strongly desired.

Source files in RWS project are currently categorized into several groups as below:

  • Include directory is used to export the UUID which is mandatory for any application build for RTL87X3E. No modification is permitted by any 3rd party.
  • Lib directory includes all binary symbol files that user application is built on.
  • Cmsis directory is used for booting up code.
  • IO directory includes all the module code that the RWS project has used.
  • Module directory includes the led module and spp_ota_service module.
  • Console directory includes the console's implementation, for example: UART.
  • CLI directory includes the cli (command-line interface) cmd, user can define themselves's, which bases on console's string mode.
  • BQB directory includes the bqb cmd for "Bluetooth Qualification Body", which bases on console's string mode.
  • BLE directory includes BLE profiles or services that the sample application has used.
  • XiaoAi directory includes the implementation of XIAOAI voice assistant.
  • XiaoWei directory includes the implementation of XIAOWEI voice assistant.
  • APP directory includes the implementation of RWS user application.
  • MP directory includes the MP test cmd, which bases on console's binary mode.
  • GFPS directory includes the implementation of gfps in application.
  • AMA directory includes the implementation of voice assistant.
  • Teams directory includes the implementation of teams specification.
  • Tuya directory includes the implementation of Tuya specification.
  • leaudio directory includes the source code of le audio related app procedure.
  • Dongle directory includes the implementation of gaming and conference dongle in application.
  • Bond directory includes the implementation of BLE bond and sync of ble bond.
  • Vendor sensor directory includes the driver of sensors.
  • USB directory includes the implementation of usb device class in application.

4 Run

4.1 EVB Wire Map

This chapter describes the hardware requirements for EVB. Certain jumpers should be connected for EVB preparation. Here take RTL8763ESE as an example to introduce the essential usage of power supply and audio interface. There may be some difference in EVB wire map for different IC (RTL8763EAU/RTL8763EFL/RTL8763EW-VC/RTL8763EWE-VP/RTL8773ESL/RTL8773DO), please refer to the respective Evaluation Board User Guide for details.

4.1.1 Power Supply

There are two power supply modes for EVB: USB connector and Li-on battery

If EVB is power supplied by the USB connector, please follow the steps below and the wire map is shown in Figure 4-1.
Step 1: Connect VBAT of jumper J38 to U_5V_FT (TP25).
Step 2: Connect AC adaptor to micro-USB connector CON3 on the opposite side of the board.

Figure 4-1 RTL87X3E USB Connector Power Supply

if EVB is power supplied by the Li-on battery, please follow the steps below and the wire map is shown in Figure 4-2.
Step 1: Short jumper J38 (VBAT and BAT+).
Step 2: Connect Li-ion battery to BAT socket J36.

Figure 4-2 RTL87X3E Li-on Battery Power Supply

4.1.2 Microphones

When MIC1 jack is used for electret condenser microphone input, please follow the steps below:
Step 1: Short J22 and J24, as shown in red rectangle box.
Step 2: Connect J25.1 with term 1(+) of microphone.
Step 3: Connect J23.1 with term 2(-) of microphone.

Figure 4-3 MIC1 Connections for Condenser Microphone

The wire map for MIC2/MIC3 jack is similar.

4.1.3 Headphone

Headphone jack is intended for headphone output. Connection of jumpers depends on application.

The following are the steps to perform single-end application:
Step 1: Disconnect J12, J8 and J13.
Step 2: Connect J9.1 with J9.2.
In this way, U3.1 is connected to ground. SPKL_P and SPKR_P are transmitted through tantalum capacitors C9 and C6 to daughterboard.

The following are the steps to perform cap-less application:
Step 1: Short J12, and connect J9.3 with J9.2. SPKL_N and SPKR_N are connected together to U3.1 for differential line.
Step 2: Short J8 and J13. SPKL_P and SPKR_P are transmitted to daughterboard directly.

Figure 4-4 Capless Connection for Headphone Out

4.3 Print Logs

After binary files have been downloaded to the chip successfully, users could reset it to run the application. And in Debug Analyzer tool, users could check whether application runs as expected by the log messages. The specific operation steps can refer to 4 DebugAnalyzer in Tool Set User Manual .

4.4 Verification Test

After completing the above steps, the verification process can begin. Two EVBs need to be near and power on at the same time to enter the RWS first engage, it is expected to hear the following voice prompt: Power on / 1 or 2(This portends first engage success)/ Pairing mode, then users can search and connect the buds by using mobile phone.

Note:

  1. BT address/Bud role/Bud side of two buds must be set different, first engage will filter the antenna RSSI from each other.
  2. The external 2.4G antenna must be installed on EVB.

5 Debug

A FT232R chip is mounted into the EVB motherboard, and it's used to convert UART to USB signals so that the developer could simply connect the EVB kit to PC via USB for logging. SWD interface on EVB motherboard would provide debug interface to J-Link for on-line debugging.

Figure 5-1 SWD Connection

J-Link could help debug and trace applications on BT Audio.

For more debugging feature introduction, Debugging System could be referred to.
