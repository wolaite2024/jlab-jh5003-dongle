◆ SDIO_GetFlagStatus()

FlagStatus SDIO_GetFlagStatus ( uint32_t  SDIO_FLAG)

Check whether the specified SDIO flag is set.

rtl876x_sdio.h

Parameters
[in]SDIO_FLAGSpecifies the SDIO flag to check. This parameter can be one of the following values:
  • SDIO_FLAG_BUF_READ_EN: Buffer read enable, this status is used for non-DMA read transfer. SET: Have data in buffer when block data is ready in the buffer. RESET: No Data in buffer when block data is ready in the buffer. Supplement: A change of this bit from 0 to 1 occurs when block data is ready in the buffer and generates the Buffer Read Interrupt.
  • SDIO_FLAG_WRITE_BUF_EN: Write buffer enable, this status is used for non-DMA write transfer. SET: Can write in non-DMA write transfer. RESET: Can not write in non-DMA write transfer. Supplement: A change of this bit from 0 to 1 occurs when top of block data can be written to the buffer and generates the Buffer Write Ready Interrupt.
  • SDIO_FLAG_READ_TF_ACTIVE: Read transfer active status. SET: Transfer data in a read transfer. RESET: No valid data in a read transfer.
  • SDIO_FLAG_WRITE_TF_ACTIVE: Write transfer active status. SET: Transfer data in a write transfer. RESET: No valid data in a write transfer.
  • SDIO_FLAG_DAT_LINE_ACTIVE: DAT line active status. SET: DAT line active. RESET: DAT line inactive.
  • SDIO_FLAG_CMD_DAT_INHIBIT: Command inhibit(DAT) status. SET: Can not issue command which uses the DAT line. RESET: Can issue command which uses the DAT line.
  • SDIO_FLAG_CMD_INHIBIT: Command inhibit(CMD) status. SET: Can not issue command which uses the CMD line. RESET: Can issue command which uses the CMD line.
Returns
The new state of SDIO_FLAG.
Return values
SETThe specified SDIO flag is set.
RESETThe specified SDIO flag is unset.

Example usage

void SD_WaitCmdLineIdle(void)
{
//Check command inhibit(CMD) status
while (SDIO_GetFlagStatus(SDIO_FLAG_CMD_INHIBIT) == SET)
{
//add user code here.
}
}

◆ SDIO_INTStatusConfig()

void SDIO_INTStatusConfig ( uint32_t  SDIO_INTStatus,
FunctionalState  newState 
)

Enable or disable the specified SDIO interrupt status.

rtl876x_sdio.h

Parameters
[in]SDIO_INTStatusSpecifies the SDIO interrupt status to be enabled or disabled. This parameter can be one of the following values:
  • SDIO_INT_ERROR: Error interrupt signal caused by any type error in error interrupt status register.
  • SDIO_INT_CARD: Card interrupt status.
  • SDIO_INT_DMA: DMA interrupt status. In case of ADMA, by setting int field in the descriptor table, host controller generates this interrupt.
  • SDIO_INT_TF_CMPL: Transfer complete interrupt status. Set when a read/write transfer and a command with busy is complete.
  • SDIO_INT_CMD_CMPL: Command complete interrupt status. Set when get the end bit of command response(Except Auto CMD12).
  • SDIO_BLOCK_GAP_EVENT: Set when stop at Block Gap Request.
[in]newStateNew state of the specified SDIO interrupt status. This parameter can be one of the following values:
  • ENABLE: Enable the specified SDIO interrupt status.
  • DISABLE: Disable the specified SDIO interrupt status.
Returns
None.

Example usage

void sdio_config(void)
{
SDIO_INTStatusConfig(SDIO_INT_CMD_CMPL, ENABLE);
}

◆ SDIO_ErrrorINTStatusConfig()

void SDIO_ErrrorINTStatusConfig ( uint32_t  SDIO_INTStatus,
FunctionalState  newState 
)

Enable or disable the specified SDIO error interrupt status.

rtl876x_sdio.h

Parameters
[in]SDIO_INTStatusSpecifies the SDIO error interrupts status to be enabled or disabled. This parameter can be one or any combination of the following values:
  • SDIO_INT_VENDOR_SPECIFIC_ERR: Additional status bits can be defined in this register by the vendor.
  • SDIO_INT_ADMA_ERR: Set when the host controller detects errors during ADMA based data transfer.
  • SDIO_INT_AUTO_CMD12_ERR: Set when detecting that one of this bits D00-D04 in Auto CMD12 Error Status register has changed from 0 to 1.
  • SDIO_INT_CURRENT_LIMIT_ERR: Current limit error, by setting the SD Bus Power bit in the Power Control register, the Host Controller is requested to supply power for the SD Bus.
  • SDIO_INT_DAT_END_BIT_ERR: Set when detecting 0 at the end bit position of read data which uses the DAT line or at the end bit position of the CRC Status.
  • SDIO_INT_DAT_CRC_ERR: Set when detecting a CRC error in data transfer through the DAT line.
  • SDIO_INT_DAT_TIMEOUT_ERR: Set when detecting one of following timeout conditions: Busy timeout for R1b,R5b type; Busy timeout after Write CRC status; Write CRC Status timeout; Read Data timeout.
  • SDIO_INT_CMD_INDEX_ERR: Set if the command index error occurs in response to a command.
  • SDIO_INT_CMD_END_BIT_ERR: Set when detecting that the end bit of command response is 0.
  • SDIO_INT_CMD_CRC_ERR: Set when detecting a CRC error in the command response.
  • SDIO_INT_CMD_TIMEOUT_ERR: Set if no response is returned within 64 SDCLK cycles from the end bit of command.
[in]newStatenew state of the specified SDIO error interrupt status. This parameter can be one of the following values:
  • ENABLE: Enable the specified SDIO error interrupt status.
  • DISABLE: Disable the specified SDIO error interrupt status.
Returns
None.

Example usage

void sdio_error_config(void)
{
SDIO_ErrrorINTStatusConfig(SDIO_INT_CMD_TIMEOUT_ERR | SDIO_INT_CMD_CRC_ERR, ENABLE);
}
