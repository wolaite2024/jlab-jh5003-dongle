99 // This describes which milestone/variant a particular range of the incoming stream is targeted for, where in the stream it is, and
100 // where it belongs on the target. Each entry requries a line in the offer file. The MCU must read this array before the main
101 // stream begins to know which part of the stream to forward, and to where in the component.
102 // Each instance of a component is a specific hardware milestone and variant. It is required that an update uniquely match only one
103 // milestone/variant for a component, i.e., a specific component must only match one line in the offer list.
