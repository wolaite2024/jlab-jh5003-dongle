1 Introduction

The purpose of this document is to give an overview of the BLE ANCS client application. ANCS is apple notification center service. It is a notification service in IOS system. After the device is connected to the ios mobile phone, the device first needs to enable the notification function of the IOS system, that is, the GATT client on the device discovers the ANCS on the IOS, and enables the notify function of its notification source characteristics. After that, the IOS system will push messages to the device through BLE.

ANCS client will be introduced according to the following several parts:

  1. How to enable ANCS client will be introduced in chapter 2 Project Overview.
  2. ANCS source code overview will be introduced in chapter 3 Source Code Overview.

2 Project Overview

2.1 Add ANCS Source Files

ANCS client related source files is ancs_client.c and ancs.c. If App want to use ANCS client, App need add those files in to keil project.

  • ancs_client.c can be find in sdk\src\ble\profile\client
  • ancs.c can be find in sdk\src\sample\ble_peripheral.

2.2 Enable ANCS Client

F_APP_BLE_ANCS_CLIENT_SUPPORT in app_flags.h shall be set to 1 to enable ANCS client function.

#define F_APP_BLE_ANCS_CLIENT_SUPPORT 1

3.3 ANCS Notification Enable

App can use function ancs_set_data_source_notify() to set the notifcation flag of data source.

T_APP_RESULT ancs_client_cb(T_CLIENT_ID client_id, uint8_t conn_id, void *p_data)
{
T_APP_RESULT result = APP_RESULT_SUCCESS;
T_ANCS_CB_DATA *p_cb_data = (T_ANCS_CB_DATA *)p_data;
switch (p_cb_data->cb_type)
{
case ANCS_CLIENT_CB_TYPE_DISC_STATE:
switch (p_cb_data->cb_content.disc_state)
{
case DISC_ANCS_DONE:
APP_PRINT_INFO0("ANCS BLE Client CB: discover procedure done.");
ancs_set_data_source_notify(conn_id, true);
break;
}
break;
}
}
