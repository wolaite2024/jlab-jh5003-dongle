Detailed Description

Add isr trace information for traceAlyzer.

traceAlyzer can not trace isr automatically. os_trace_isr_create() give a name to the isr. os_trace_isr_begin() & os_trace_isr_end() can determine the trace scope.

bool(* os_trace_isr_create) (void **pp_handle, const char *p_name, uint32_t priority)

Stores a name and priority level for an Interrupt Service Routine, to allow for better visualization. Returns a handle used by os_trace_isr_begin.

os_trace.h

Parameters
[out]pp_handleUsed to pass back a handle by which the isr can be traced.
[in]p_nameA descriptive name for the isr.
[in]prioritythe hardware priority of the isr.
Returns
The status of the isr trace creation.
Return values
trueISR trace handle was created successfully.
falseISR trace handle was failed to create.

Example usage

#define PRIO_ISR_TIMER1 3
void *p_handle = NULL;
int test(void)
{
// Create a isr trace
if (os_trace_isr_create(&p_handle, "isr name", PRIO_ISR_TIMER1) == true)
{
// isr_trace created successfully.
}
else
{
// isr_trace failed to create.
return -1;
}
return 0;
}
void ISR_handler()
{
os_trace_isr_begin(p_handle);
...
os_trace_isr_end();
}
