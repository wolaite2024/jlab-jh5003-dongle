◆ t_bt_a2dp_lhdc_min_bitrate

enum t_bt_a2dp_lhdc_min_bitrate

BT A2DP LHDC minimum bitrate.

Enumerator
BT_A2DP_LHDC_MIN_BITRATE_AUTO_ADJUST_LOW 

No lower limit, but re-adjusted by specific rules: 44.1/48KHz: 64kbps (re-adjusted to at least 64Kbps); 96/192KHz: 256kbps (re-adjusted to at least 256Kbps).

BT_A2DP_LHDC_MIN_BITRATE_AUTO_ADJUST_HIGH 

No lower limit, but re-adjusted by specific rules: 44.1/48KHz: 128kbps (re-adjusted to at least 128Kbps); 96/192KHz: 256kbps (re-adjusted to at least 256Kbps).

BT_A2DP_LHDC_MIN_BITRATE_256KBPS 

Minimum bitrate 256Kbps.

BT_A2DP_LHDC_MIN_BITRATE_400KBPS 

Minimum bitrate 400Kbps.

◆ bt_a2dp_connect_cfm()

bool bt_a2dp_connect_cfm ( uint8_t  bd_addr[6],
uint16_t  mtu_size,
bool  accept 
)

Send an A2DP connection confirmation.

Parameters
[in]bd_addrRemote BT address.
[in]mtu_sizeLocal AVDTP stream channel MTU size.
[in]acceptAccept or reject the connection indication.
  • true Accept the connection indication.
  • false Reject the connection indication.
Note
If the application cannot choose an appropriate value for mtu_size, or if there are no specific requirements for the value of mtu_size, it can be set to 0. Here, 0 means that the A2DP profile will select an appropriate value (typically 675) for the application. The value of mtu_size will be ignored if the parameter accept is set to false.
Returns
The status of sending the A2DP connection confirmation.
Return values
trueA2DP connection confirmation was sent successfully.
falseA2DP connection confirmation was failed to send.
