IO_MSG handle API

APP will use this following API to handle IO_MSG. All the application events are pre-handled in this function. All the io messages are sent to this function. Then the event handling function shall be called according to the message type.

void app_io_handle_msg(T_IO_MSG io_driver_msg_recv)
{
uint16_t msgtype = io_driver_msg_recv.type;
switch (msgtype)
{
case IO_MSG_TYPE_GPIO:
{
switch (io_driver_msg_recv.subtype)
{
case IO_MSG_GPIO_KEY:
app_key_handle_msg(&io_driver_msg_recv);
break;
case IO_MSG_GPIO_UART_WAKE_UP:
app_transfer_handle_msg(&io_driver_msg_recv);
break;
case IO_MSG_GPIO_CHARGER:
app_charger_handle_msg(&io_driver_msg_recv);
break;
case IO_MSG_GPIO_USB_AUDIO_PLUG:
app_usb_audio_plug_handle_msg(&io_driver_msg_recv);
break;
