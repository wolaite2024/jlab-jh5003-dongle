Detailed Description

Create, control and destroy Audio Track sessions.

Each Audio Track represents an audio stream path from the low level hardware, which shall be unidirectional or bidirectional for the input (record) or output (playback) streams, and be attached to the audio devices. The application can pass data in and drain data out by writing to and reading from the Audio Track, or adjust the gain level by controlling the Audio Track volumes.

typedef enum t_audio_track_policy T_AUDIO_TRACK_POLICY

Define Audio Track policy.

audio_track.h

Audio Track can set its stream policy. AUDIO_TRACK_POLICY_SINGLE_STREAM is the default policy that only one running Audio Track instance for either decoding path or encoding path. The Audio Track instance with stream type AUDIO_STREAM_TYPE_VOICE will preempt the Audio Track instance with stream type AUDIO_STREAM_TYPE_PLAYBACK. The Audio Track instance with stream type AUDIO_STREAM_TYPE_VOICE cannot co-exist with the Audio Track instance with stream type AUDIO_STREAM_TYPE_RECORD. AUDIO_TRACK_POLICY_MULTI_STREAM is the policy that multi Audio Track instances for either decoding path or encoding path can co-exist. Application can control each Audio Track instance's lifecycle individually.

Define Audio Track async read/write IO operations.

audio_track.h

Each Audio Track session can be unidirectional or bidirectional. when the stream type is set as AUDIO_STREAM_TYPE_PLAYBACK in audio_track_create, P_AUDIO_TRACK_ASYNC_IO can be implemented as an async write operation; when the stream type is set as AUDIO_STREAM_TYPE_RECORD, P_AUDIO_TRACK_ASYNC_IO can be implemented as an async read operation; when the stream type is set as AUDIO_STREAM_TYPE_VOICE, P_AUDIO_TRACK_ASYNC_IO can be implemented as either an async write or an async read respectively.

Note
If P_AUDIO_TRACK_ASYNC_IO is set as NULL in audio_track_create, the Audio Track shall use the sync read interface audio_track_read to drain data from, or use the sync write interface audio_track_write to pass data in.
Parameters
[in]handleThe Audio Track session handle T_AUDIO_TRACK_HANDLE.
[in,out]timestampThe timestamp of the buffer passed into or passed out from Audio Track.
[in,out]seq_numThe sequence number of the buffer passed into or passed out from Audio Track.
[in,out]statusThe frame status of the buffer passed into or passed out from Audio Track.
[in,out]frame_numThe frame number of the buffer passed into or passed out from Audio Track.
[in,out]bufThe buffer that holds the audio data written to or read from the Audio Track.
[in]required_lenThe required size in bytes written to or read from the Audio Track.
[out]actual_lenThe actual size in bytes written to or read from the Audio Track.

Create an Audio Track session instance.

audio_track.h

Audio system may have multiple coexisting or exclusive low-level audio data paths; if the application wants to control the specific audio data path, it should first create an Audio Track that is abstract from the low-level hardware device routing. The operations upon Audio Track shall focus on the corresponding Audio Track handle T_AUDIO_TRACK_HANDLE.

Note
This is an async function, and the Audio Track session state will be transformed into the created state when the current state AUDIO_TRACK_STATE_CREATED in Audio event AUDIO_EVENT_TRACK_STATE_CHANGED is received.
Parameters
[in]stream_typeThe stream type T_AUDIO_STREAM_TYPE of the Audio Track.
[in]modeThe stream mode T_AUDIO_STREAM_MODE of the Audio Track.
[in]usageThe stream usage T_AUDIO_STREAM_USAGE of the Audio Track.
[in]format_infoThe encoding/decoding format information T_AUDIO_FORMAT_INFO of the Audio Track.
[in]volume_outThe stream volume out level of the Audio Track, and the range is between audio_volume_out_min_get and audio_volume_out_max_get.
[in]volume_inThe stream volume in level of the Audio Track, and the range is between audio_volume_in_min_get and audio_volume_in_max_get.
[in]deviceThe device bitmask Audio Device Bitmask of the Audio Track.
[in]async_writeThe async write operation P_AUDIO_TRACK_ASYNC_IO of the Audio Track.
[in]async_readThe async read operation P_AUDIO_TRACK_ASYNC_IO of the Audio Track.
Returns
The instance handle of Audio Track session. If returned handle is NULL, the Audio Track session instance was failed to create.

◆ audio_track_start()

bool audio_track_start ( T_AUDIO_TRACK_HANDLE  handle)

Start playing the Audio Track session.

audio_track.h

Audio Track shall be started before writing to or reading from the Audio Track session, otherwise, data may be dropped for writing or be empty for reading. When the Audio Track session is stopped by audio_track_stop, this API can resume the Audio Track session.

Note
This is an async function, and the Audio Track session state will be transformed into the started state when the current state AUDIO_TRACK_STATE_STARTED in Audio event AUDIO_EVENT_TRACK_STATE_CHANGED is received. Starting an Audio Track session already in started state will be ignored.
Parameters
[in]handleThe Audio Track session handle T_AUDIO_TRACK_HANDLE.
Returns
The status of starting the Audio Track.
Return values
trueAudio Track was started successfully.
falseAudio Track was failed to start.

Stop the Audio Track session.

audio_track.h
