T_GAP_CAUSE le_aox_set_conn_cte_receive_params ( uint8_t  conn_id,

Set Connection CTE Receive Parameters. Result will be returned by app_gap_aox_callback with cb_type GAP_MSG_LE_AOX_SET_CONN_CTE_RECEIVE_PARAMS.

Parameters
[in]conn_idConnection ID.
[in]sampling_enableEnable/disable Connection IQ sampling: T_GAP_AOX_SAMPLING_ENABLE_TYPE.
[in]slot_durationsSwitching and sampling slots: T_GAP_SLOT_DUATIONS_TYPE.
[in]switching_pattern_lengthThe number of Antenna IDs in the pattern. Range: 0x02 to max_switching_pattern_length supported by controller returned by le_aox_read_antenna_information.
[in]p_antenna_idsAntenna ID in the pattern.
Returns
Result of sending request.
Return values
GAP_CAUSE_SUCCESSSend request success.
GAP_CAUSE_SEND_REQ_FAILEDSend request sent fail.
GAP_CAUSE_NON_CONNFailed. No connection.
GAP_CAUSE_INVALID_PARAMFailed. Invalid parameter.

Example usage

void test(void)
{
uint8_t conn_id = 0;
T_GAP_AOX_SAMPLING_ENABLE_TYPE sampling_enable = GAP_AOX_SAMPLING_ENABLE;
T_GAP_SLOT_DUATIONS_TYPE slot_durations = GAP_SLOT_DURATIONS_SWITCH_SAMPLE_2US;
uint8_t switching_pattern_length = 2;
uint8_t p_antenna_ids[2] = {0, 1};
T_GAP_CAUSE cause = le_aox_set_conn_cte_receive_params(conn_id, sampling_enable, slot_durations,
switching_pattern_length, p_antenna_ids);
}

◆ le_aox_conn_cte_request_enable()

T_GAP_CAUSE le_aox_conn_cte_request_enable ( uint8_t  conn_id,
T_GAP_AOX_CTE_REQUEST_ENABLE_TYPE  enable,
uint16_t  cte_request_interval,
uint8_t  requested_cte_length,
T_GAP_CTE_TYPE  requested_cte_type 
)

Request the Controller to start or stop initiating the Constant Tone Extension Request procedure. Result will be returned by app_gap_aox_callback with cb_type GAP_MSG_LE_AOX_CONN_CTE_REQUEST_ENABLE.

IQ samples will be returned by app_gap_aox_callback with cb_type GAP_MSG_LE_AOX_CONN_IQ_REPORT_INFO on Constant Tone Extensions received. Otherwise, request failed info will be returned by app_gap_aox_callback with cb_type GAP_MSG_LE_AOX_CTE_REQUEST_FAILED_INFO.

Parameters
[in]conn_idConnection ID.
[in]enableEnable/disable Constant Tone Extension Request: T_GAP_AOX_CTE_REQUEST_ENABLE_TYPE. If Enable is set to GAP_AOX_CTE_REQUEST_DISABLE, the remaining parameters shall be ignored.
[in]cte_request_intervalDefines whether the Constant Tone Extension Request procedure is initiated only once or periodically.
  • 0x0000: Initiate the Constant Tone Extension Request procedure once, at the earliest practical opportunity.
  • 0x0001 to 0xFFFF: Requested interval for initiating the Constant Tone Extension Request procedure in number of connection events.
[in]requested_cte_lengthThe minimum length of the Constant Tone Extension that the Controller shall request from the remote device.
  • Range: 0x02 to 0x14.
  • Units: 8 us.
[in]requested_cte_typeThe type of Constant Tone Extension that the Controller shall request from the remote device: T_GAP_CTE_TYPE.
Returns
Result of sending request.
Return values
GAP_CAUSE_SUCCESSSend request success.
GAP_CAUSE_SEND_REQ_FAILEDSend request sent fail.
GAP_CAUSE_NON_CONNFailed. No connection.
GAP_CAUSE_INVALID_PARAMFailed. Invalid parameter.

Example usage

void test(void)
{
uint8_t conn_id = 0;
T_GAP_AOX_CTE_REQUEST_ENABLE_TYPE enable = GAP_AOX_CTE_REQUEST_ENABLE;
uint16_t cte_request_interval = 0;
uint8_t requested_cte_length = 2;
T_GAP_CTE_TYPE requested_cte_type = GAP_CTE_TYPE_AOA;
T_GAP_CAUSE cause = le_aox_conn_cte_request_enable(conn_id, enable, cte_request_interval,
requested_cte_length, requested_cte_type);
}
