Glossary

1 Introduction

Specific service UUID option is used when the headset wants to be bonded to a particular mobile application.
Both the headset and the mobile application need to assign this service UUID for bonding check.

2 McuConfigTool Configuration

Specific service UUID option can be enabled in Vendor UUID within BR/EDR page.

Figure 2-1 SPP Specific service UUID


Figure 2-2 LE Specific service UUID
  • This option is enabled when the checkbox is checked.
  • For SPP application, 16-bytes specific UUID can be filled in the input box for SDP handshake or RTK-defined SPP SDP UUID is used.
  • For LE application, first 15 of 16-bytes specific UUID is used for advertising data or RTK-defined LE service UUID is used.

3 Source Code Overview

This section describes how to assign specific service UUID.
The reference files are shown as follows:

3.1 Initiate SPP connection from Source

Source code directory: sdk/src/sample/rws/app_sdp.c
SPP specific UUID is filled into SDP table to be discovered by Source side.

#if F_APP_SPECIFIC_UUID_SUPPORT
if (app_cfg_const.enable_specific_service_uuid)
{
memcpy(&spp_specific_uuid_sdp_record[8], app_cfg_const.specific_service_uuid, 16);
bt_sdp_record_add((void *)spp_specific_uuid_sdp_record);
}
else
{
bt_sdp_record_add((void *)rtk_vendor_spp_sdp_record);
}
#else
bt_sdp_record_add((void *)rtk_vendor_spp_sdp_record);
#endif

3.2 Initiate SPP connection from headset

Source code directory: sdk/src/sample/rws/app_spp.c
SPP specific UUID is registered via bt_spp_service_register.
Headset will compare the registered SPP UUID with Source's SPP UUID which is discovered via SDP.

#if F_APP_SPECIFIC_UUID_SUPPORT
if (app_cfg_const.enable_specific_service_uuid)
{
bt_spp_service_register((uint8_t *)app_cfg_const.specific_service_uuid,
RFC_RTK_VENDOR_CHANN_NUM);
}
else
{
bt_spp_service_register((uint8_t *)rtk_vendor_spp_service_class_uuid128, RFC_RTK_VENDOR_CHANN_NUM);
}
#else
bt_spp_service_register((uint8_t *)rtk_vendor_spp_service_class_uuid128, RFC_RTK_VENDOR_CHANN_NUM);
#endif

3.3 Init LE advertising data

Source code directory: sdk/src/sample/rws/app_ble_common_adv.c
First 15 of 16-bytes specific UUID is filled into advertising data to be scanned by Central.

#if F_APP_SPECIFIC_UUID_SUPPORT
if (app_cfg_const.enable_specific_service_uuid)
{
memcpy(&common_adv_data[5], app_cfg_const.specific_service_uuid, 15);
}
#endif
