◆ shm2_io_ctl()

uint32_t shm2_io_ctl ( uint32_t  ioctl,
uint32_t  para 
)

To get informations about DSP2 or register callbacks to communicate with application.

Parameters
ioctldefined in T_SHM2_IO_CTL
SHM2_IOCTL_VERSION: Get shm2 fw version
SHM2_IOCTL_CALLBACK_CMD_EVENT: Register callback to process command received from dsp2
SHM2_IOCTL_CALLBACK_RX_REQ: Register callback to process data received from dsp2
SHM2_IOCTL_CALLBACK_TX_ACK: Register callback to process ACK from dsp2
SHM2_IOCTL_CALLBACK_MAILBOX: Register callback to process mailbox event from dsp2
parafor SHM2_IOCTL_VERSION, the actual type of is (uint32_t*) to get fw version
for SHM2_IOCTL_CALLBACK_CMD_EVENT, the actual type of is SHM2_IRQ_CB. The para will be called when DSP2 has sent a command to mcu by command queue.
SHM2_IOCTL_CALLBACK_RX_REQ, the actual type of is SHM2_IRQ_CB. The para will be called when DSP2 has sent data to mcu by data queue indexed by T_D2H_DATA_Q_ID.
SHM2_IOCTL_CALLBACK_TX_ACK, the actual type of is SHM2_IRQ_CB. The para will be called when DSP2 has received data from mcu by data queue indexed by T_H2D_DATA_Q_ID.
SHM2_IOCTL_CALLBACK_MAILBOX, the actual type of is SHM2_MAILBOX_CB. The para will be called when mcu receive mailbox interrupt from DSP2. The group parameter in SHM2_MAILBOX_CB is defined in T_MAILBOX_EVT
Note
: callbacks indexed by SHM2_IOCTL_CALLBACK_CMD_EVENT/SHM2_IOCTL_CALLBACK_RX_REQ/SHM2_IOCTL_CALLBACK_TX_ACK/SHM2_IOCTL_CALLBACK_MAILBOX are called in ISR.
Returns
uint32_t 0: success other: fail

◆ shm2_cmd_send()

bool shm2_cmd_send ( uint8_t *  buffer,
uint16_t  length,
bool  flush 
)

To send command to DSP2.

Parameters
buffercontent of command
lengthlength of command
flushIf command has been sent after the function returns.
true: command has been put into cmd queue and inform DSP2 about the cmd.
false: command has just put into shm queue
Returns
true command is sent successfully
false command is sent unsuccessfully
Note
the API SHOULD be called AFTER MAILBOX_EVT_SHM2_READY in T_MAILBOX_EVT
before communication between mcu and dsp2, engine_start SHOULD be called to prevent dsp from power-down
Example
#include "shm2_api.h"
#include "audio_probe.h"
void mailbox_cb(uint32_t evt, uint32_t param)
{
switch (evt)
{
case MAILBOX_EVT_SHM2_READY:
{
send msg IPC_READY to IPC_task
}
break;
default:
break;
}
}
void IPC_task()
{
shm2_io_ctl(SHM2_IOCTL_CALLBACK_MAILBOX, mailbox_cb);
receive msg
{
if IPC_READY
{
engine_start();
uint8_t cmd[64] = {};
shm2_cmd_send(cmd, sizeof(cmd), true);
}
}
}
