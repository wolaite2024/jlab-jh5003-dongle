◆ le_past_recipient_set_default_periodic_adv_sync_transfer_params()

T_GAP_CAUSE le_past_recipient_set_default_periodic_adv_sync_transfer_params ( T_GAP_PAST_RECIPIENT_PERIODIC_ADV_SYNC_TRANSFER_PARAM *  p_periodic_adv_sync_transfer_param)

Specify the initial value for PAST recipient periodic advertising sync transfer parameter to be used for all subsequent connections. NOTE: This function does not affect any existing connection.

NOTE: This function can be called after stack is ready. Explanation: If stack is ready, Application will be notified by message GAP_MSG_LE_DEV_STATE_CHANGE with new_state about gap_init_state which is configured as GAP_INIT_STATE_STACK_READY.

If sending request operation is success, the result of setting parameters will be returned by app_gap_callback with cb_type GAP_MSG_LE_PAST_RECIPIENT_SET_DEFAULT_PERIODIC_ADV_SYNC_TRANSFER_PARAMS.

Reception of periodic advertising synchronization information will be returned by app_gap_callback with cb_type GAP_MSG_LE_PAST_RECIPIENT_PERIODIC_ADV_SYNC_TRANSFER_RECEIVED_INFO. GAP PA synchronization states will be returned by app_gap_callback with cb_type GAP_MSG_LE_PA_SYNC_STATE_CHANGE_INFO. Periodic advertisement will be returned by app_gap_callback with cb_type GAP_MSG_LE_PERIODIC_ADV_REPORT_INFO.

Parameters
[in]p_periodic_adv_sync_transfer_paramPAST recipient periodic advertising sync transfer parameter. T_GAP_PAST_RECIPIENT_PERIODIC_ADV_SYNC_TRANSFER_PARAM.
Returns
Send request operation.
Return values
GAP_CAUSE_SUCCESSSend request operation success.
OthersSend request operation failure.
