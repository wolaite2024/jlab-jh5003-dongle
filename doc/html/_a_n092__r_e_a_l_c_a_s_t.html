LEA LE Audio

1 Introduction

The purpose of this document is to introduce Realcast resolution for broadcast audio and how to use Realcast. Realcast is a fresh approach suggested by Realtek based on the development of the Public Broadcast Profile(PBP). Realcast Source, Realcast Sink, and Realcast Assistant make up its three roles, which are comparable to PBP Source, PBP Sink, and PBP Assistant.

2 Current Situation

Currently, LE Audio PBP experiences the following problems:

  1. PBP Assistant needs to be based on Android 13 or later, and must support PA/PAST.
    Table 1-1 Android Support LEA
    Android PA PAST LE Audio Assistant API
    13 Yes Yes Yes
    12 Yes Yes No
    11 No No No
  2. PBP employs Broadcast Name to differentiate between various broadcast devices, but this is insufficient to separate the content delivered by several PBP sources.
  3. The 16-byte binary numbers that make up PBP's broadcast code are not user-friendly enough.

3 Improving Resolution

  1. The following activities can be carried out by Realcast Assistant using the GATT-based services that Realcast Sink offers:
    a) Realcast Assistant instructs Realcast Sink to look for nearby broadcast information and notify Realcast Assistant of the findings.
    b) The PBP Broadcast Code can be entered by Realcast Assistant after choosing the desired broadcast from the list.
    c) Realcast Sink synchronizes to the appropriate broadcast source using Realcast Assistant's choice.
  2. In order to make sure that every device is distinct, Realcast Source adds the Broadcast Device information while broadcasting the Broadcast Name.
  3. Realcast Source exposes information about its broadcast content through QR code, which may contain the following data:
    a) Broadcast Name (must be supported)
    b) Broadcast Code (must be supported)
    c) Device information (Realcast must support, standard PBP may or may not support)
    d) User information (Realcast must support, standard PBP may or may not support)
    e) Manufacturer information (Realcast must support, standard PBP may or may not support
    f) QR Code encoding version information (Realcast must support, standard PBP may not support)

4 Realcast Source

The purpose of this chapter is to provide an overview of the equipment needed to serve as Realcast Source. Generally, RTL87X3E and mobile phones, televisions are able to become Realcast Source if they have the following features.

Public Broadcast Announcement

PBP's original AUX_ADV_IND packet contains:
1. Public Broadcast Announcement
2. Broadcast Name AD Type
Realcast needs to add more content of device information in the following format to better distinguish source devices:

Table 4-1 Realcast Information
Parameters Value Description
Length
Sum of all subsequent data length
AD Type 0xFF Fixed field, 1 byte
Realcast Command ID 0x5D,0x00,0x0E Fixed field, 3 byte
Realcast Vendor 0x00,0x00 Default: 0x00, 0x00.
Manufacturers can apply to Realtek PM if they want to stand out from the competition
Realcast Device When Realcast Vendor value is set to 0x0000, Realcast Device length is 6 bytes, and it matches the manufacturer's BT device's Public Address(Little Endian).
When Realcast Vendor is set to other values, the manufacturer defines it themselves.

Broadcast Code Create

  1. The system automatically generates randomly
  2. A 16-byte hexadecimal number is entered by the user. For example, 0011223344556677889900AABBCCDDEEFF
  3. Enter characters that are visible, transform them to 16-bit hexadecimal values, and then encode the data using the UTF8 encoding. If the length is insufficient, add 0x00 to the platform. The platform must have a method for updating the Broadcast Code.

Broadcast Code Distribute

Considering that the Broadcast Code is not user-friendly enough, it is recommended to prioritize QR Code distribution

Figure 1-1 QR Code

The format of QR Code content is as follows:
http://bt.realtek.com/realcast?v=1&name=Realcast111&code=0011223344556677889900AABBCCDDEEFF&device=0000FFEEDDCCBBAA&vendor=rtk&uid=1234567890
1. http://bt.realtek.com/realcast? : Fixed prefix
2. v=1: Realcast version number
3. name=Realcast111: PBP Broadcast Name
4. Subgroups=1: Indicates the number of SubGroups be used for this broadcast in the BIG. N=1~0xFF denotes the existence of SubGroups, while n=0 denotes that there are none known and that it is up to the headphones to find them. When this field is absent, it means that there is just one subgroup.
5. code=0011223344556677889900AABBCCDDEEFF : PBP Broadcast Code(When a TV customer requests a manual password input, this field may not be included, so Assistant must pop up an interface for the customer to input)
6. Device=0000FFEEDDCCBBAA: Device code, which in the Public Broadcast Announcement relates to the Realcast Vendor and Realcast Device. The Realcast Vendor is 0x0000, and the Realcast Device is FF, EE, DD, CC, BB, AA, according to the information above. Realcast must provide this field; otherwise, it will be regarded as a Bluetooth SIG standard PBP.
7. Vendor=rtk: This is the product's manufacturer; it is optional information. Realcast Assistant doesn't presently use it, but future cloud services will use it
8. UID=1234567890: This is the product's user UID; it is optional. Realcast Assistant doesn't presently use it, but future cloud services will.

In order to make it easier for everyone to choose the appropriate Realcast Assistant for installing headphones, Realcast must also be labeled around the QR Code. It is also required to synchronously display the name and code on the Realcast Source UI(this field might not be visible when TV consumers need to manually enter their passwords), making it simple for other PBP assistants to manually earn money.

5 Realcast Sink

Realtek vendor advertising is now used by Realcast Sink to enable Realcast assistants to find and establish connections. The service UUID looks like this:
0x12, 0xA2, 0x4D, 0x2E, 0xFE, 0x14, 0x48, 0x8E, 0x93, 0xD2, 0x17, 0x3C, 0xFD, 0x02, 0x00, 0x00

6 Realcast Assistant

The purpose of this chapter is to provide an overview of the equipment needed to serve as Realcast Assistant and how to use Realcast. Generally, mobile phones and televisions are able to become Realcast Assistant which intalls the APK(https://www.pgyer.com/rtk_realcast_assistant_a).

My Devices

The user-configured Sink devices will be shown on the My Device page. To connect instantly, click to pick a device.

Figure 6-1 My Devices

Connect Sink

The page will fly to the device details page after selecting the Sink device, and the program will immediately establish a BLE connection with the device. A blank page will be shown following a successful connection, necessitating the manual inclusion of the Source device.

Figure 6-2 Connect Sink

Delete Sink

The device will be eliminated from my device list and any locally saved sink and source data will also be erased by clicking the 'Delete' button.

Figure 6-3 Delete Sink
