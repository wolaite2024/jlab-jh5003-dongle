◆ P_AUDIO_PLUGIN_ACTION_HANDLER

typedef void(* P_AUDIO_PLUGIN_ACTION_HANDLER) (T_AUDIO_PLUGIN_HANDLE handle, T_AUDIO_PLUGIN_PARAM param, void *context)

Define the Audio Plugin action handler prototype.

Each Audio Plugin instance has the capability to perform customized actions at specific occasions as defined by T_AUDIO_PLUGIN_OCCASION. When creating an Audio Plugin instance using audio_plugin_create, the action handler within T_AUDIO_PLUGIN_POLICY is registered to Audio Plugin.

Note
The Audio Plugin action is always treated as an asynchronous procedure, regardless of its implementation details. This means that the Audio Plugin cannot determine when the action is completed until the application returns the same context through the audio_plugin_msg_send interface.
Parameters
[in]handleThe Audio Plugin instance handle T_AUDIO_PLUGIN_HANDLE.
[in]paramThe parameter for the Audio Plugin action handler.
[in]contextThe context for the Audio Plugin instance to use in audio_plugin_msg_send.
