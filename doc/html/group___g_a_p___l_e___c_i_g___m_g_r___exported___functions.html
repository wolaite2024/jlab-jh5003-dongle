◆ cig_mgr_setup_data_path()

T_GAP_CAUSE cig_mgr_setup_data_path ( uint16_t  cis_conn_handle,
uint8_t  path_direction,
uint8_t  path_id,
uint8_t  codec_id[5],
uint32_t  controller_delay,
uint8_t  codec_len,
uint8_t *  p_codec_data 
)

Identify and create the isochronous data path for CIS.

Note
This function is used for ISOCH_ROLE_INITIATOR and ISOCH_ROLE_ACCEPTOR.

If sending request operation is success, the result of creating isochronous data path will be returned by app_isoc_cis_initiator_cig_cb for ISOCH_ROLE_INITIATOR or app_isoc_cis_acceptor_cb for ISOCH_ROLE_ACCEPTOR with cb_type MSG_CIG_MGR_SETUP_DATA_PATH.

Parameters
[in]cis_conn_handleConnection handle of the CIS.
[in]path_directionSpecify the direction for which the data path is being configured. LE Isochronous Channels Definition Setup Data Path Data Path Direction Flag.
[in]path_idSpecify the data transport path used.
  • 0x00: HCI. DATA_PATH_ID_HCI.
  • 0x01 to 0xFE: Logical_Channel_Number. The meaning of the logical channel is vendor-specific.
[in]codec_idSpecify the coding format used over the air.
  • Octet 0: See Assigned Numbers for Coding Format.
  • Octets 1 to 2: Company ID, see Assigned Numbers for Company Identifier. Shall be ignored if octet 0 is not 0xFF.
  • Octets 3 to 4: Vendor-defined codec ID. Shall be ignored if octet 0 is not 0xFF.
[in]controller_delayController delay in microseconds.
  • Range: 0x000000 to 0x3D0900.
  • Time range: 0 s to 4 s.
[in]codec_lenLength of codec configuration.
[in]p_codec_dataSpecify codec-specific configuration information for the specified direction.
Returns
Send request operation.
Return values
GAP_CAUSE_SUCCESSSend request operation success.
OthersSend request operation failure.
