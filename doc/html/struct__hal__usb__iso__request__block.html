Detailed Description

USB isochronous request block.

For isochronous transfer, ping-pong buffers are used to transfer data, for example, if current data transfer uses buf0,
the next data transfer will use buf1, and vice versa. Data process interval depends on both bInterval in endpoint descriptor and buf_proc_intrvl: 2^(bInterval - 1) * buf_proc_intrvl *125us

Parameters
buf0buffer 0 of ping-pong buffers
buf1buffer 1 of ping-pong buffers
data_per_framedata length per host polling
buf_proc_intrvlnumber of host polling per data process
pkt_cntpacket number per data process
iso_pkt0packets information stored in buf0, refer to T_HAL_USB_ISO_PKT_INFO
iso_pkt1packets information stored in buf1, refer to T_HAL_USB_ISO_PKT_INFO
ep_handlehandle of endpoint that will transfer data
completethis callback will be called when data that has been sent or received.
For sending data, upper application SHOULD prepare data in buf indexed by proc_buf_num.
For receiving data, upper application SHOULD process data stored in buf indexed by proc_buf_num..
