1 Introduction

This document is used to introduce how to use Xiaoai related functions in RWS applications. To use the Xiao Ai service, the 'Xiao Ai Tong Xue' application should be installed on user's phone. The app is available on both Android and iOS. Users can install it from corresponding application store of the phone.

2 Configuration

2.1 Configuration of MCU Config Tool

To use Xiao Ai service, users need to do some configuration on MCU Config Tool. Firstly, the XiaoAi option in third party page should be checked. After that, users need to configure the VID, PID, Major id and Minor id values, which are assigned by MIUI. The LE option and RFCOMM option are both needed to be checked. The ADV timeout value defines the duration of ble advertising.

Figure 2-1 Mcu Config Tool

In order to test the voice assistant conveniently, the "XiaoAi wakeword" function to trigger the voice assistant needs to be mapped to a key on the page of KEY (Figure 2-2).

3 Pairing

3.1 Android

The device will enable advertising after power on. Users could find the device in 'Xiao Ai Tong Xue' by clicking the button 'Add Device'. Choose the device and a popup window for pairing will appear. Click the 'Pair' button and the device state will be Connected.

Figure 3-1 Add Device on Android App

3.2 iOS

On iOS, Users could find the device like Android in 'Xiao Ai Tong Xue' by clicking the button 'Add Device'. Choose the device and the app will ask you to pair with the target device on the settings page. Go to settings-bluetooth and pair with the device. Return to 'Xiao Ai Tong Xue' app and a popup window for pairing will appear. Click the 'Pair' button and the device state will be Connected.

4 OTA

Users are able to update the images over the air (OTA). Xiao Ai OTA function is only supported on test version of 'Xiao Ai Tong Xue' app currently. To get the test app, please contact RTK FAE.

5 Common Code Flow

5.1 Device Discovery

By sending advertising packets with specific format, the device could be discovered by 'Xiao Ai Tong Xue'. The format of advertising packets is defined by MIUI.

The advertising API are as follows:

bool le_xm_xiaoai_adv_start(uint16_t timeout_sec) //Start Xiao Ai advertising
bool le_xm_xiaoai_adv_stop(void); //Stop Xiao Ai advertising

5.2 Voice Assistant

Voice Assistant State (VA_STATE) has 5 sub-states including idle state, active pending state, active state, cancel pending state and invalid state.

typedef enum
{
VA_STATE_IDLE = 0,
VA_STATE_ACTIVE_PENDING = 1,
VA_STATE_ACTIVE = 2,
VA_STATE_CANCEL_PENDING = 3,
VA_STATE_INVALID = 4
} T_VA_STATE;

5.2.1 Start Voice Assistant

By mapping "XiaoAi wakeword" function to a key in MCU Config Tool, Xiao Ai Voice Assistant can be waken up by pressing corresponding key.

#if XM_XIAOAI_FEATURE_SUPPORT
case MMI_XIAOAI_WAKEWORD:
{
APP_PRINT_INFO1("app_handle_mmi_message: MMI_XIAOAI_WAKEWORD bud_role %d", app_cfg_nv.bud_role);
if (app_cfg_nv.bud_role != REMOTE_SESSION_ROLE_SECONDARY && app_db.xiaoai_support)
{
uint8_t app_idx = app_a2dp_get_active_idx();
app_xiaoai_device_va_start(app_db.br_link[app_idx].bd_addr, true, false);
}
}
break;
#endif

The API for start voice assistant is :

bool app_xiaoai_device_va_start(uint8_t *bd_addr, bool from_local,
bool accept_start_speech_from_remote)

If there exists a BLE link or an RFCOMM link with authentication, the device will send the command of starting voice assistant to the mobile phone. The VA_STATE will switch to VA_STATE_ACTIVE_PENDING from VA_STATE_IDLE and the device will wait for response of starting voice assistant from remote. The API for handling response of starting voice assistant is:

void app_xm_handle_cmd_rsp_speech_start(T_XM_CMD_COMM_WAY_TYPE type, uint8_t *bd_addr, uint8_t flag)
{
app_xiaoai_start_recording(bd_addr);
app_xiaoai_device_set_va_state(bd_addr, VA_STATE_ACTIVE);
}

The device will start recording and the VA_STATE will switch to VA_STATE_ACTIVE after receiving response of starting voice assistant.

The voice data is sent by API:

bool xm_cmd_speech_data(T_XM_CMD_COMM_WAY_TYPE type, uint8_t *bd_addr, uint8_t conn_id,
uint8_t *p_data, uint16_t len);

5.2.2 Stop Voice Assistant

The app will judge whether the recording is over or not. If the recording is over, it will send command to device to stop voice assistant. The API for handling command of stop voice assistant is:

void app_xm_handle_cmd_rsp_speech_stop(T_XM_CMD_COMM_WAY_TYPE type, uint8_t *bd_addr)
{
app_xiaoai_stop_recording(bd_addr);
app_xiaoai_device_set_va_state(bd_addr, VA_STATE_IDLE);
}
The device will stop recording and the VA_STATE will switch to VA_STATE_IDLE after receiving command
of stopping voice assistant.

5.3 XiaoAi OTA

The 'Xiao Ai Tong Xue' app will send start OTA command to the device as if user press the button 'Start OTA'. The API to do Xiao Ai OTA is:

void xiaoai_ota_handle_cmd(T_XM_CMD_COMM_WAY_TYPE type, uint8_t *bd_addr, uint8_t conn_id,
uint8_t opcode,
bool req_rsp, uint8_t *p_data, uint16_t len);

Commands used in the OTA process are as follows:

XM_OPCODE_DEV_REBOOT = 0x03,
...
XM_OPCODE_OTA_GET_DEVICE_UPDATE_FILEINFO_OFFSET = 0xE1,
XM_OPCODE_OTA_INQUIRE_DEVICE_IF_CAN_UPDATE = 0xE2,
XM_OPCODE_OTA_ENTER_UPDATE_MODE = 0xE3,
XM_OPCODE_OTA_EXIT_UPDATE_MODE = 0xE4,
XM_OPCODE_OTA_SEND_FIRMWARE_UPDATE_BLOCK = 0xE5,
XM_OPCODE_OTA_GET_DEVICE_REFRESH_FIRMWARE_STATUS = 0xE6,
XM_OPCODE_OTA_NOTIFY_UPDATE_MODE = 0xE7,

Firstly, the Phone should ask for the update file information from the device. Secondly, the Phone will send the header of update file to device. The device will check the header and response to the phone if the device could update. After that, the device will enter update mode.

In the process of update, the primary device is updated first, and then the secondary device is updated. the update of secondary device adopts relay mode that the primary device receives the data and transmits the data to the secondary.
