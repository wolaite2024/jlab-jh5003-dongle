◆ T_GPIO_TYPE

enum T_GPIO_TYPE
Enumerator
GPIO_TYPE_AUTO 

The GPIO function switch automatically, use core GPIO in active mode, use aon GPIO in low power mode.

GPIO_TYPE_CORE 

The GPIO function in core domain, only works when active mode. Support all the features.

GPIO_TYPE_AON 

The GPIO function in aon domain, works in dlps/power down, much slower than CORE mode, not support read level and debounce.

GPIO_TYPE_AUTO 

The GPIO function switch automatically, use core GPIO in active mode, use aon GPIO in low power mode.

GPIO_TYPE_CORE 

The GPIO function in core domain, only works when active mode. Support all the features.

GPIO_TYPE_AON 

The GPIO function in aon domain, works in DLPS/power down, much slower than CORE mode. Reading level and interrupt are not supported for GPIO_TYPE_AON. If gpio type set to GPIO_TYPE_AON in input mode, hal_gpio will switch to GPIO_TYPE_AUTO.

◆ hal_gpio_init_pin()

T_GPIO_STATUS hal_gpio_init_pin ( uint8_t  pin_index,
T_GPIO_TYPE  type,
T_GPIO_DIRECTION  direction,
T_GPIO_PULL_VALUE  pull_value 
)

Initialize a pin to gpio mode.

hal_gpio.h

Parameters
[in]pin_indexThe pin index, please refer to rtl876x.h "Pin_Number" part.
[in]typeThe TYPE of GPIO mode to be used.
[in]directionThe direction GPIO is set to, could be output mode or input mode.
[in]pull_valueThe GPIO pull value is set to , could be pull high low or none.
Returns
The status of the gpio pin initialization.
Return values
GPIO_STATUS_OKThe GPIO pin was initialized successfully.
GPIO_STATUS_ERROR_PINThe GPIO pin was failed to initialized due to invalid pin number.

Example usage

int test(void)
{
hal_gpio_init_pin(TEST_PIN, GPIO_TYPE_CORE, GPIO_DIR_OUTPUT ,GPIO_PULL_UP );
hal_gpio_init_pin(TEST_PIN_2, GPIO_TYPE_CORE, GPIO_DIR_INPUT ,GPIO_PULL_UP);
}
