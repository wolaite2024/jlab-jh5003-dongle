Initializes the KeyScan peripheral according to the specified parameters in the KeyScan_InitStruct.

rtl876x_keyscan.h

Parameters
[in]KeyScanSelected KeyScan peripheral, which can be KEYSCAN.
[in]KeyScan_InitStructPointer to a KEYSCAN_InitTypeDef structure that contains the configuration information for the specified KeyScan peripheral.
Returns
None.

Example usage

void driver_keyscan_init(void)
{
RCC_PeriphClockCmd(APBPeriph_KEYSCAN, APBPeriph_KEYSCAN_CLOCK, ENABLE);
KEYSCAN_InitTypeDef KEYSCAN_InitStruct;
KeyScan_StructInit(&KEYSCAN_InitStruct);
KEYSCAN_InitStruct.rowSize = KEYBOARD_ROW_SIZE;
KEYSCAN_InitStruct.colSize = KEYBOARD_COLUMN_SIZE;
KEYSCAN_InitStruct.scanmode = KeyScan_Manual_Scan_Mode;
KEYSCAN_InitStruct.debounceEn = vDebounce_En;
KeyScan_Init(KEYSCAN, &KEYSCAN_InitStruct);
}

◆ KeyScan_INTConfig()

void KeyScan_INTConfig ( KEYSCAN_TypeDef *  KeyScan,
uint32_t  KeyScan_IT,
FunctionalState  newState 
)

Enable or disable the specified KeyScan interrupts.

rtl876x_keyscan.h

Parameters
[in]KeyScanSelected KeyScan peripheral, which can be KEYSCAN.
[in]KeyScan_ITSpecifies the KeyScan interrupts sources to be enabled or disabled. This parameter can be any combination of the following values:
  • KEYSCAN_INT_OVER_READ: When there is no data in the FIFO, reading the FIFO will trigger this interrupt to prevent over-reading.
  • KEYSCAN_INT_THRESHOLD: This interrupt is triggered when data in the FIFO reaches the threshold level.
  • KEYSCAN_INT_SCAN_END: No matter whether the key value is scanned or not, the interrupt will be triggered as long as the scanning action is completed.
  • KEYSCAN_INT_FIFO_NOT_EMPTY: If there is data in FIFO, the interrupt will be triggered.
  • KEYSCAN_INT_ALL_RELEASE: When the release time cnt reaches the set value, if no key is pressed, the interrupt is triggered.
[in]newStateNew state of the specified KeyScan interrupts. This parameter can be one of the following values:
  • ENABLE: Enable the specified KeyScan interrupts.
  • DISABLE: Disable the specified KeyScan interrupts.
Returns
None.

Example usage

void driver_keyscan_init(void)
{
KeyScan_INTConfig(KEYSCAN, KEYSCAN_INT_SCAN_END | KEYSCAN_INT_ALL_RELEASE, ENABLE);
}

Enable or disable the specified KeyScan interrupts.

Parameters
KeyScanselected KeyScan peripheral.
KeyScan_ITspecifies the KeyScan interrupts sources to be enabled or disabled. This parameter can be any combination of the following values:
  • KEYSCAN_INT_TIMEOUT: KeyScan timeout interrupt mask
  • KEYSCAN_INT_OVER_THRESHOLD: Kescan FIFO data over threshold interrupt mask
  • KEYSCAN_INT_SCAN_END: KeyScan scan end interrupt mask
NewStatenew state of the specified KeyScan interrupts. This parameter can be: ENABLE or DISABLE.
Return values
None

uint32_t  KeyScan_FLAG 
)

Clear the specified KeyScan flags.

rtl876x_keyscan.h

Parameters
[in]KeyScanSelected KeyScan peripheral, which can be KEYSCAN.
[in]KeyScan_FLAGSpecifies the KeyScan flag to clear. This parameter can be one of the following values:
  • KEYSCAN_FLAG_FIFOLIMIT: Set when the keylimit is set to 1, press two buttons and only send one button.
  • KEYSCAN_FLAG_DATAFILTER: Set when FIFO data filter bit is normal.
  • KEYSCAN_FLAG_OVR: Set when the fifo is full, there is new data sends to the fifo.
Returns
None.
Note
KEYSCAN_FLAG_FULL and KEYSCAN_FLAG_EMPTY can't be cleared manually. They are cleared by hardware automatically.

Example usage

void keyscan_demo(void)
{
KeyScan_ClearFlags(KEYSCAN, KEYSCAN_FLAG_FIFOLIMIT);
}
