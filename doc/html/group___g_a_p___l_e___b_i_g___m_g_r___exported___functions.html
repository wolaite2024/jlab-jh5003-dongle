T_GAP_CAUSE gap_big_mgr_setup_data_path ( uint16_t  bis_conn_handle,
uint8_t  data_path_direction,

uint8_t  codec_id[5],
uint32_t  controller_delay,
uint8_t  codec_config_len,
uint8_t *  p_codec_config 
)

Identify and create the isochronous data path for BIS.

NOTE: This function is used for BIG_MGR_ROLE_ISOC_BROADCASTER and BIG_MGR_ROLE_SYNC_RECEIVER.

If sending request operation is success, the result of creating isochronous data path will be returned by app_isoc_bis_broadcaster_cb for BIG_MGR_ROLE_ISOC_BROADCASTER or app_isoc_bis_receiver_cb for BIG_MGR_ROLE_SYNC_RECEIVER with cb_type MSG_BIG_MGR_SETUP_DATA_PATH.

Parameters
[in]bis_conn_handleConnection handle of the BIS.
[in]data_path_directionSpecify the direction for which the data path is being configured. LE Isochronous Channels Definition Setup Data Path data path direction flag. BIG_MGR_ROLE_ISOC_BROADCASTER uses DATA_PATH_ADD_INPUT. BIG_MGR_ROLE_SYNC_RECEIVER uses DATA_PATH_ADD_OUTPUT.
[in]data_path_idSpecify the data transport path used. 0x00: HCI. DATA_PATH_ID_HCI. 0x01 to 0xFE: Logical_Channel_Number. The meaning of the logical channel is vendor-specific.
[in]codec_idSpecify the coding format used over the air. Octet 0: See Assigned Numbers for Coding Format Octets 1 to 2: Company ID, see Assigned Numbers for Company Identifier. Shall be ignored if octet 0 is not 0xFF. Octets 3 to 4: Vendor-defined codec ID. Shall be ignored if octet 0 is not 0xFF.
[in]controller_delayController delay in microseconds. Range: 0x000000 to 0x3D0900 Time range: 0 s to 4 s
[in]codec_config_lenLength of codec configuration.
[in]p_codec_configSpecify codec-specific configuration information for the specified direction.
Returns
Send request operation.
Return values
GAP_CAUSE_SUCCESSSend request operation success.
OthersSend request operation failure.

Example usage

void test(void)
{
T_GAP_CAUSE cause = gap_big_mgr_setup_data_path(bis_conn_handle, data_path_direction, data_path_id, codec_id, controller_delay, codec_config_len, p_codec_config);
}
