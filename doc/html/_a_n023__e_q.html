Introduction

EQ (equalizer) is used to adjust the volume of different frequency bands within an audio signal. EQ module provides the functionalities to control audio software EQ effect easily. The EQ mentioned below refers to audio software EQ.

The types of EQ

There are two types of EQ, speaker EQ and microphone EQ.

Speaker EQ

  • Speaker EQ is been used when playing music. There are four modes of speaker EQ, line-in EQ, gaming EQ, ANC EQ and normal EQ.
  • Line-in EQ, gaming EQ and ANC EQ are been designed for the line-in scenario, gaming mode and ANC mode separately.
  • The EQ modes mechanism is described in EQ modes mechanism.

Microphone EQ

  • The only one mode of microphone EQ is APT mode.
  • The APT mode means the EQ of DSP APT. LLAPT does not support EQ feature.

How to configure EQ

DSPConfigTool

  • Speaker EQ

    • Checking "Effect #0" in Audio page to enable speaker EQ.
    • Clicking "Parametric EQ" in Audio page to open setting page.
      Figure 2: Enable the Speaker EQ
    • Audio Spk Parametric EQ Setting
      1. Global Gain
        • The setting of global gain.
      2. Application
        • The four options represent normal EQ, gaming EQ, ANC EQ and Line-in EQ separately. The options share all setting except IsEnable and IsDefault fields.
      3. StageNum
        • The number of stage of the EQ.
        • Adding the number of stage will increate the size of DSP config image. The size of a stage is about 45 bytes.
      4. IsEnable
        • In current EQ mode, which EQ index are available.
        • For gaming/ANC/line-in EQ, it allows that no checking any EQ index. It will been regarded as disable gaming/ANC/line-in EQ feature.
      5. IsDefault
        • In current EQ mode, which EQ index is the default.
      6. Freq, Gain, Q and Type
        • The setting for current EQ.
      7. Total Response
        • The wave of current setting.
    Figure 3: Audio Spk Parametric EQ Setting
  • DSP APT EQ
    • The way to open setting page is Audio page -> Enable normal APT -> APT Process -> MIC EQ Setting.
      Figure 4: DSP APT EQ
    • The setting is the same as speaker EQ.
      Figure 5: Audio Mic Parametric EQ Setting

MCU config tool

  • MMI
    • If the number of EQ is greater than 1, SoC can switch EQ via MMI.
    • "Audio EQ Switch (0x6B)" is been used to switch speaker EQ. "APT EQ Switch (0x91)" is been used to switch APT EQ.
      Figure 6: The key MMI
  • Ringtone
    • When switching EQ via MMI or AudioConnect, SoC plays ringtone to notice user.
      Figure 7: The ringtone of audio EQ
      Figure 8: The ringtone of APT EQ
  • Reset EQ index when power on
    • If checking this option, SoC resets EQ index to default EQ when power on.
      Figure 9: Reset EQ index when power on
  • RWS separate adjust apt eq by audio connect
    • This option is for RWS product.
    • It allows adjusting APT EQ of left earbud and right earbud separately via AudioConnect.
      Figure 10: RWS separate adjust apt eq
  • The number of user EQ
    • This options are setting how many the number of adjusted speaker/APT EQ can been saved in SoC.
      • For example, if setting the number of speaker EQ is six, the first six speaker EQ in DSPConfigTool can been saved in SoC.
        Figure 11: User EQ

EQ modes mechanism

Speaker EQ

  • EQ mode changed
    • When enter/exit the special mode, SoC applies corresponding EQ without EQ ringtone automatically.
      • For example, SoC applies gaming EQ when enter gaming mode. And applies normal EQ when exit gaming mode.
    • When SoC in multiple modes, the priority of EQ from high to low is line-in EQ, gaming EQ, ANC EQ and normal EQ.
      • For example, SoC applies gaming EQ when both gaming mode and ANC are been enabled.
    • If the corresponding EQ are not been setting,it will been regarded as disable gaming/ANC/line-in EQ feature.
      • For example, if no any IsEnable field of gaming EQ been checked, the EQ will not change when enter/exit gaming mode.
    • The code to decide EQ mode is as below.
      uint8_t app_eq_judge_audio_eq_mode(void)
      {
      uint8_t new_eq_mode = NORMAL_MODE;
      #if F_APP_LINEIN_SUPPORT
      if (app_line_in_plug_state_get() && (eq_utils_num_get(SPK_SW_EQ, LINE_IN_MODE) != 0))
      {
      new_eq_mode = LINE_IN_MODE;
      }
      else
      #endif
      {
      if (app_db.gaming_mode && (eq_utils_num_get(SPK_SW_EQ, GAMING_MODE) != 0))
      {
      new_eq_mode = GAMING_MODE;
      }
      #if F_APP_ANC_SUPPORT
      else if (app_anc_is_anc_on_state(app_db.current_listening_state)
      && (eq_utils_num_get(SPK_SW_EQ, ANC_MODE) != 0))
      {
      new_eq_mode = ANC_MODE;
      }
      #endif
      else
      {
      new_eq_mode = NORMAL_MODE;
      }
      }
      return new_eq_mode;
      }
  • Last used EQ index
    • SoC records the last used EQ index. This EQ will been applies automatically next time.
      • For example, user switchs to third EQ. SoC will applies third EQ until user change to other EQ.
    • For multiple EQ modes, the last used EQ index are been recorded separately. When EQ mode been changed, SoC will apply separate EQ index. The variable are as below.
      eq_idx_line_in_mode_record
      eq_idx_gaming_mode_record
      eq_idx_anc_mode_record
      eq_idx_normal_mode_record
    • When user controls EQ index via MMI or cmd set, SoC only changes the EQ index of current EQ mode.

Microphone EQ

  • Due to only one mode for microphone EQ, SoC doesn't consider the EQ mode switch of microphone EQ.

How to control EQ

MMI

  • If the number of EQ of current EQ mode is greater than 1, user can switch EQ index via MMI.

AudioConnect(Command set)

  • In the setting page, user can click the "Equalizer" to control speaker EQ, and click "Real Hearing Enhancement" to control APT EQ.
    Figure 12: AudioConnect setting page
  • Speaker EQ
    • User can click the below EQ to switch to other EQ index.
    • User can adjust the value of gain of current EQ. The other parameter like freq, gain, Q and type are the same as the setting in dsp config image.
      • The way to get the EQ parameter is described in The adjusted EQ parameter.
    • When SoC is in special EQ mode, AudioConnect updates the information automatically.
      Figure 13: In different EQ mode, AudioConnect Equalizer page
  • APT EQ
    • In "Real Hearing Enhancement" page click "APT EQ" to control APT EQ.
      Figure 14: Real Hearing Enhancement
    • The way to control EQ is the same as audio EQ.
      • The way to get the EQ parameter is described in The adjusted EQ parameter.
    • If checking "RWS separate adjust apt eq by audio connect", there are LEFT and RIGHT options.
      Figure 15: AudioConnect APT EQ

The adjusted EQ parameter

  • User can adjust the gains of EQ via AudioConnect.
  • According SoC capacity and setting, the adjusted EQ parameter may been saved in phone or SoC.
  • If the adjusted EQ parameter been saved in SoC, SoC can get the parameter at any time, otherwise SoC needs reporting the EQ index to asking AudioConnect sending the adjusted EQ parameter to SoC.
  • If the adjusted EQ parameter been saved in phone, AudioConnect needs keeping running in background.
    • If AudioConnect is been terminated, the EQ effect will back to original setting.

The command set about EQ

Command

  • The EQ command is as below.
    typedef enum
    {
    ....
    CMD_AUDIO_EQ_QUERY = 0x0200,
    ....
    CMD_AUDIO_EQ_PARAM_SET = 0x0203,
    CMD_AUDIO_EQ_PARAM_GET = 0x0204,
    CMD_AUDIO_EQ_INDEX_SET = 0x0205,
    CMD_AUDIO_EQ_INDEX_GET = 0x0206,
    ....
    CMD_APT_EQ_INDEX_SET = 0x020B,
    CMD_APT_EQ_INDEX_GET = 0x020C,
    ....
    CMD_AUDIO_EQ_QUERY_PARAM = 0x0210,
    ....
    CMD_RESET_EQ_DATA
    } T_CMD_ID;

Event

  • The EQ event is as below.
    typedef enum
    {
    EVENT_ACK = 0x0000,
    ....
    EVENT_AUDIO_EQ_REPLY = 0x0200,
    EVENT_AUDIO_EQ_PARAM_REPORT = 0x0201,
    EVENT_AUDIO_EQ_INDEX_REPORT = 0x0202,
    ....
    EVENT_APT_EQ_INDEX_INFO = 0x0206,
    EVENT_AUDIO_EQ_REPLY_PARAM = 0x0208,
    ....
    } T_EVENT_ID;
  • EVENT_AUDIO_EQ_INDEX_REPORT
    • SoC reports this event with below parameter.
      typedef enum
      {
      EQ_INDEX_REPORT_BY_PLAYING,
      EQ_INDEX_REPORT_BY_CHANGE_MODE,
      EQ_INDEX_REPORT_BY_SWITCH_EQ_INDEX,
      EQ_INDEX_REPORT_BY_GET_EQ_INDEX,
      EQ_INDEX_REPORT_BY_UPDATE_EQ_INDEX,
      EQ_INDEX_REPORT_BY_GET_UNSAVED_EQ,
      } T_EQ_DATA_UPDATE_EVENT;
    • If the adjusted EQ parameter saves in phone. SoC reports below parameter to get the EQ parameter.
      • EQ_INDEX_REPORT_BY_PLAYING
        • SoC sends this parameter when the music played. SoC will expect the response about current EQ.
      • EQ_INDEX_REPORT_BY_CHANGE_MODE
        • SoC sends this parameter when the EQ mode changed. SoC will expect the response about current EQ.
      • EQ_INDEX_REPORT_BY_SWITCH_EQ_INDEX
        • SoC sends this parameter when EQ switched via MMI. SoC will expect the response about current EQ.
    • If the adjusted EQ parameter saves in SoC.
      • When EQ mode/index changed, SoC reports EQ_INDEX_REPORT_BY_UPDATE_EQ_INDEX let AudioConnect to update UI.
      • When RWS engaged, SoC reports EQ_INDEX_REPORT_BY_GET_UNSAVED_EQ to get unsaved EQ parameter.
    • EQ_INDEX_REPORT_BY_GET_EQ_INDEX
      • SoC sends this parameter when receiving CMD_AUDIO_EQ_INDEX_GET.
  • EVENT_APT_EQ_INDEX_INFO
    typedef enum
    {
    EQ_INDEX_REPORT_BY_APT_ENABLE,
    EQ_INDEX_REPORT_BY_RWS_CONNECTED,
    EQ_INDEX_REPORT_BY_SWITCH_APT_EQ_INDEX,
    EQ_INDEX_REPORT_BY_GET_APT_EQ_INDEX,
    EQ_INDEX_REPORT_BY_UPDATE_APT_EQ_INDEX,
    EQ_INDEX_REPORT_BY_GET_UNSAVED_APT_EQ,
    } T_APT_EQ_DATA_UPDATE_EVENT;
    • If the adjusted EQ parameter saves in phone. SoC reports below parameter to get the EQ parameter.
      • EQ_INDEX_REPORT_BY_APT_ENABLE
        • SoC sends this parameter when DSP APT enabled. SoC will expect the response about current EQ.
      • EQ_INDEX_REPORT_BY_RWS_CONNECTED
        • SoC sends this parameter when the EQ mode changed. SoC will expect the response about current EQ.
      • EQ_INDEX_REPORT_BY_SWITCH_APT_EQ_INDEX
        • SoC sends this parameter when EQ switched via MMI. SoC will expect the response about current EQ.
    • If the adjusted EQ parameter saves in SoC.
      • When EQ index changed, SoC reports EQ_INDEX_REPORT_BY_UPDATE_APT_EQ_INDEX let AudioConnect to update UI.
      • When RWS engaged, SoC reports EQ_INDEX_REPORT_BY_GET_UNSAVED_APT_EQ to get unsaved EQ parameter.
    • EQ_INDEX_REPORT_BY_GET_APT_EQ_INDEX
      • SoC sends this parameter when receiving CMD_APT_EQ_INDEX_GET.

API

Framework APIs

  • eq_create
    • Create an EQ instance.
    • Usage example:
      eq_instance = eq_create(eq_content_type, dynamic_eq_buf, eq_len);
  • eq_release
    • Release the specific EQ instance.
    • Usage example:
      // speaker EQ
      eq_release(p_link->eq_instance);
      // APT EQ
      eq_release(app_db.apt_eq_instance);
  • eq_enable
    • Enable the specific EQ instance.
    • Usage example:
      eq_enable(app_db.apt_eq_instance);
  • eq_disable
    • Disable the specific EQ instance.
    • Usage example:
      eq_disable(app_db.apt_eq_instance);
  • audio_track_effect_attach
    • Attach the Audio Effect instance to the specific Audio Track session.
    • Usage example:
      audio_track_effect_attach(p_link->a2dp_track_handle, p_link->eq_instance);
  • audio_passthrough_effect_attach
    • Attach the Audio Effect instance to the Audio Pass-through stream.
    • Usage example:
      audio_passthrough_effect_attach(app_db.apt_eq_instance);
  • line_in_effect_attach
    • Attach the Audio Effect instance to the line-in stream.
    • Usage example:
      line_in_effect_attach(app_db.line_in_eq_instance);
  • audio_track_effect_detach
    • Detach the Audio Effect instance from the specific Audio Track session.
    • Usage example:
      audio_track_effect_detach(p_link->a2dp_track_handle, p_link->eq_instance);
  • audio_passthrough_effect_detach
    • Detach the Audio Effect instance from the Audio Pass-through stream.
    • Usage example:
      audio_passthrough_effect_detach(app_db.apt_eq_instance);
  • line_in_effect_detach
    • Detach the Audio Effect instance from the line-in stream.
    • Usage example:
      line_in_effect_detach(app_db.line_in_eq_instance);
  • eq_set
    • Set the specific EQ instance band parameter.
    • Usage example:
      eq_set(link->eq_instance, dynamic_eq_buf, eq_len);

APP EQ module APIs

  • app_eq_init
    • Init APP EQ module.
    • Usage example:
      app_eq_init();
  • app_eq_deinit
    • Deinit APP EQ module.
    • Usage example:
      app_eq_deinit();
  • app_eq_create
    • Create EQ instance.
    • Usage example:
      // speaker EQ
      p_link->eq_instance = app_eq_create(EQ_CONTENT_TYPE_AUDIO, SPK_SW_EQ, app_db.spk_eq_mode,
      app_cfg_nv.eq_idx);
      // APT EQ
      app_db.apt_eq_instance = app_eq_create(EQ_CONTENT_TYPE_PASSTHROUGH, \
      MIC_SW_EQ, APT_MODE, app_cfg_nv.apt_eq_idx);
  • app_eq_index_set
    • Set the specific EQ index.
    • Usage example:
      // speaker EQ
      app_eq_index_set(SPK_SW_EQ, app_db.spk_eq_mode, app_cfg_nv.eq_idx);
      // APT EQ
      app_eq_index_set(MIC_SW_EQ, APT_MODE, app_cfg_nv.apt_eq_idx);
  • app_eq_param_get
    • Get specific EQ parameter.
    • Usage example:
      // get the EQ parameter sended to DSP
      eq_len = app_eq_param_get(eq_type, mode, index, dynamic_eq_buf,
      app_db.max_eq_len, EQ_DATA_TO_DSP, sample_rate);
      // get the EQ parameter sended to source
      eq_len = app_eq_param_get((T_EQ_TYPE)p_eq_info->sw_eq_type, p_eq_info->eq_mode, p_eq_info->eq_idx,
      eq_data_temp, app_db.max_eq_len, EQ_DATA_TO_PHONE, sample_rate);
  • app_eq_param_set
    • Set the specific EQ index parameter.
    • Usage example:
      app_eq_param_set(eq_idx, &cmd_ptr[0], eq_len_to_dsp);
  • app_eq_play_audio_eq_tone
    • Play audio EQ tone.
    • Usage example:
      app_eq_play_audio_eq_tone();
  • app_eq_play_apt_eq_tone
    • Play APT EQ tone.
    • Usage example:
      app_eq_play_apt_eq_tone();
  • app_eq_report_eq_param
    • Report EQ parameter.
    • Usage example:
      app_eq_report_eq_param(cmd_path, app_idx);
  • app_eq_report_terminate_param_report
    • Terminate report EQ parameter.
    • Usage example:
      app_eq_report_terminate_param_report(cmd_path, app_idx);
  • app_eq_sample_rate_get
    • Get current sample rate.
    • Usage example:
      uint32_t sample_rate = app_eq_sample_rate_get();
  • app_eq_idx_check_accord_mode
    • Check idx accord EQ mode.
    • Usage example:
      app_eq_idx_check_accord_mode();
  • app_eq_sync_idx_accord_eq_mode
    • Check EQ mode then sync idx
    • Usage example:
      app_eq_sync_idx_accord_eq_mode(eq_mode, eq_idx);
  • app_eq_cmd_handle
    • EQ related command handler.
    • Usage example:
      app_eq_cmd_handle(cmd_ptr, cmd_len, cmd_path, app_idx, ack_pkt);
  • app_eq_audio_eq_enable
    • Enable the EQ effect.
    • Usage example:
      app_eq_audio_eq_enable(&p_link->eq_instance, &p_link->audio_eq_enabled);
  • app_eq_change_audio_eq_mode
    • Change audio speaker EQ mode.
    • Usage example:
      app_eq_change_audio_eq_mode(true);
  • app_eq_judge_audio_eq_mode
    • Judge audio speaker EQ mode.
    • Usage example:
      app_db.spk_eq_mode = app_eq_judge_audio_eq_mode();
  • app_eq_save_user_eq_to_ftl
    • Save specific user EQ to FTL.
    • Usage example:
      app_eq_save_user_eq_to_ftl((T_EQ_TYPE)app_db.sw_eq_type, eq_mode, eq_idx, eq_data[0],
      eq_adjust_side,
      &eq_data[1], eq_len_to_dsp,
      &eq_data[1 + eq_len_to_dsp],
      p_audio_eq_info->eq_data_len - eq_len_to_dsp - 1);
  • app_eq_is_valid_user_eq_index
    • Check the specific EQ is valid user EQ or not.
    • Usage example:
      #if (F_APP_USER_EQ_SUPPORT == 1)
      if (app_eq_is_valid_user_eq_index(SPK_SW_EQ, app_db.spk_eq_mode, app_cfg_nv.eq_idx))
      {
      app_report_eq_idx(EQ_INDEX_REPORT_BY_GET_UNSAVED_EQ);
      }
      else
      #endif
      {
      app_report_eq_idx(EQ_INDEX_REPORT_BY_PLAYING);
      }
  • app_eq_reset_user_eq
    • Reset specific user EQ.
    • Usage example:
      app_eq_reset_user_eq((T_EQ_TYPE)eq_type, eq_mode, eq_idx);
  • app_eq_reset_all_user_eq
    • Reset all user EQ.
    • Usage example:
      app_eq_reset_all_user_eq();
  • app_eq_reset_unsaved_user_eq
    • Reset unsaved user EQ.
    • Usage example:
      app_eq_reset_unsaved_user_eq();
  • app_eq_sync_user_eq_when_connected
    • Sync user EQ when bud connected.
    • Usage example:
      app_eq_sync_user_eq_when_connected();
  • app_eq_continue_sync_user_eq_when_connected
    • Continue sync user EQ when bud connected.
    • Usage example:
      app_eq_continue_sync_user_eq_when_connected(false, eq_type, eq_index, offset, len - 5);
  • app_eq_process_sync_user_eq_when_b2b_connected
    • Process sync user EQ message when bud connected.
    • Usage example:
      app_eq_process_sync_user_eq_when_b2b_connected(eq_type, eq_index, offset, &buf[5], len - 5);
  • app_eq_report_sec_eq_to_pri
    • Report the EQ of secondary to primary.
    • Usage example:
      app_eq_report_sec_eq_to_pri(eq_type, eq_mode, eq_index, cmd_path, app_idx);
  • app_eq_report_sec_eq_to_src
    • Report the EQ of secondary to source.
    • Usage example:
      app_eq_report_sec_eq_to_src(cmd_path, app_idx, eq_len, &buf[7]);
