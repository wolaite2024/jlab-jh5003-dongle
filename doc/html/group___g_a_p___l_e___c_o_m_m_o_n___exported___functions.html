◆ le_gap_msg_info_way()

void le_gap_msg_info_way ( bool  use_msg)

Set gap message inform way.

Default value is true.

  1. When use_msg is true, gap will send the gap message to io_queue registered by gap_start_bt_stack. Message type is GAP BT Message Type Definitions.
  2. When use_msg is false, gap will send the gap message using callback function registered by le_register_app_cb. Message type is GAP_MSG_LE_GAP_STATE_MSG.
Parameters
[in]use_msgWhether to use message.
Return values
voidExample usage
int test(void)
{
le_gap_msg_info_way(false);
}

◆ le_set_gap_param()

T_GAP_CAUSE le_set_gap_param ( T_GAP_LE_PARAM_TYPE  param,
uint8_t  len,
void *  p_value 
)

Set a GAP Common parameter.

NOTE: You can call this function with a GAP Common Parameter ID and it will set the GAP Parameter. GAP Peripheral Parameters are defined in (gap.h). If the "len" field sets to the size of a "uint16_t" ,the "p_value" field must point to a data with type of "uint16_t".

Parameters
[in]paramProfile parameter ID: T_GAP_LE_PARAM_TYPE
[in]lenLength of data to write
[in]p_valuePointer to data to write. This is dependent on the parameter ID and WILL be cast to the appropriate data type (For example: if data type param is uint16, p_value will be cast to pointer of uint16_t).
Returns
Set result
Return values
GAP_CAUSE_SUCCESSSet parameter success.
otherSet parameter failed.

Example usage

void app_le_gap_init(void)
{
...
//device name and device appearance
uint8_t device_name[GAP_DEVICE_NAME_LEN] = "BLE_PERIP";
uint16_t appearance = GAP_GATT_APPEARANCE_UNKNOWN;
//Set device name and device appearance
le_set_gap_param(GAP_PARAM_DEVICE_NAME, GAP_DEVICE_NAME_LEN, device_name);
le_set_gap_param(GAP_PARAM_APPEARANCE, sizeof(appearance), &appearance);
...
}

◆ le_get_gap_param()

T_GAP_CAUSE le_get_gap_param ( T_GAP_LE_PARAM_TYPE  param,
void *  p_value 
)

Get a GAP Common parameter.

NOTE: You can call this function with a GAP Common Parameter ID and it will get a GAP Common Parameter. GAP Common Parameters are defined in (gap.h). Also, the "p_value" field must point to a "uint16".

Parameters
[in]paramProfile parameter ID: T_GAP_LE_PARAM_TYPE
[in,out]p_valuePointer to location to get the parameter value. This is dependent on the parameter ID and will be cast to the appropriate data type (For example: if data type of param is uint16_t, p_value will be cast to pointer of uint16_t).
Returns
Get result
Return values
GAP_CAUSE_SUCCESSGet parameter success.
otherGet parameter failed.

Example usage

void test(void)
{
uint8_t addr_type = GAP_REMOTE_ADDR_LE_PUBLIC;
uint8_t addr[GAP_BD_ADDR_LEN] = {0};
//get connected device address and address type
le_get_gap_param(GAP_PARAM_LATEST_CONN_BD_ADDR_TYPE, &addr_type);
le_get_gap_param(GAP_PARAM_LATEST_CONN_BD_ADDR, &addr);
}
