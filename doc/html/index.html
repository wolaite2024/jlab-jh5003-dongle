1 Overview

In this document, we will introduce how the powerful Realtek RTL8763E series platform provides the capabilities to enrich our developers' applications. The Realtek platform offers high computation performance with ultra-low system power consumption. The rich I/O interfaces can connect almost all external devices. The high-performance/good compatibility Bluetooth interface lets the application connect to the world through a wireless connection, allowing for endless possibilities. You can find more attractive application cases from the Realtek official website https://www.realmcu.com/.

1.1 The Benefit of the Realtek solution

Realtek platform provides the SoC platform with the below features

  1. Rich and highly compatible peripheral interfaces for the IoT and bt audio enabled device
  2. High performance with low power consumption computation environment
  3. Rich development documentation to break the barrier between Realtek and the developer
  4. Mature mass production experience to help developers s build up the production line

4 RTL87x3E Software Development Guide

The Realtek SoC provides rich development resources to help developers familiarize themselves with the RTL87x3E series and start the actual development process quickly. In the below sections, we will take the RTL8763ESE as an example to illustrate our document structure.

Note: In the below sections, we can reference the link to get summary information from each document.

4.1 Product Brief

This document mainly introduces the typical features and application scenarios of the dual-mode Bluetooth microcontroller (Dual mode BT MCU) solution. Three types of applications are introduced, which are the audio transmitter, audio receiver, and audio transceiver. In this document, we offer an HW features list, SW features list, supported Bluetooth profiles, audio codec types, and audio routing capability that might help you to determine a suitable solution for your application.

4.2 SDK Quick Start Guide

This document is a short guide to help beginners quickly set up and use the EVB development board. It briefly goes over important pins on the EVB, how to generate and download images onto the EVB, print logs, and what other documents you might find important. This should be the first document that you start with if you are completely new.

4.3 SDK Resources

4.3.1 SDK User Guide

This document goes over the basics of the SDK that we provide. It describes the setup, the architecture, setting up the application and settings, downloading, and debugging. Overall this document aims to help users gain an understanding of how to use the SDK to set up projects.

4.3.2 System Introduction

4.3.2.1 DLPS (Deep Low Power State)

The user guide aims to provide users with a comprehensive understanding of the DLPS features and how to use them to maximize battery life. It describes how the DLPS works, how to enter and exit it, and important related APIs.

4.3.2.2 Memory

This document describes the memory system of the IC and introduces how to use them. The memory consists of ROM, RAM, external SPI Flash, and eFuse. Cache has dedicated RAM, and the dedicated RAM also can be configured as general RAM using Realtek Vendor registers. This flexible memory configuration mechanism lets the IC support a wide range of applications whose memory usage is different.

4.3.2.3 Flash

Flash is a kind of non-volatile storage for code shadowing to RAM, executing code directly, or storing data. This document describes how to use Flash-related APIs and how Flash works in the IC.

4.3.2.4 Security Mechanism

This paper introduces the security mechanism of the IC as well as its usage. This document guides you on how to enable non-secure IC as secure IC, generate rsa signature key, and prepare rsa signed images for the secure IC. If no security is enabled in your products, please skip this document.

4.3.2.5 DFU/ OTA (Device Firmware Upgrade Over The Air)

OTA (Over The Air) represents the technology that uses Bluetooth to update the image (code and data) that runs in the Flash. This document goes over how to use OTA, describing things such as the flash layout, the image header format, and OTA protocols.

4.3.2.6 Debug System

Debugging is the process of finding and resolving bugs within computer programs, software, or systems. Two debugging methods are provided to debug user applications in SDK: log mechanism and SWD debugging.

4.4 Audio

4.4.1 Audio Subsystem

This document provides information about the design philosophy and operating princple of Audio Subsystem. Audio Subsystem is an audio-specific domain architecture that standardizes the audio driver abstraction interfaces, virtualize the audio stream routing mechanism, and provides various high level modularized components.

4.4.2 Audio Plugin

This document provides information about the usage of Audio Plugin module. Audio Plugin is designed for the requirments that need to integrate external audio devices (vendor hardware Codecs, vendor DSPs, DACs, Amplifiers, etc.) into Audio Subsystem. Customers can use Audio Plugin module to implement cooperations between customized external audio devices and SoC internal devices.

4.4.3 Audio Route

The purpose of this document is to describe how to use the audio route module in the MCUConfig Tool. The audio route capability of each IC can be demonstrated using the MCUConfig Tool. The customers can configure the routing relationship of each audio stream by using MCUConfig Tool.

4.5 Peripheral

This document describes the technical software details for the different peripherals supported by the IC. These are things such as data structures used by the peripherals and the library functions that are supported.

4.6 BLE Stack

The Realtek Software Development Kit (SDK) provides software documentation including stack/profiles, reference material, example profiles, and user applications, aiming to help with product development using the Realtek Series of System on Chip (SOC) devices. The purpose of this document is to give an overview of BLE Stack Interfaces. BLE Stack Interfaces can be divided into Generic Access Profile (GAP) interfaces and Generic Attribute Profile (GATT) based profile interfaces.

4.7 BREDR Stack

The purpose of this document is to give an overview of Bluetooth BR/EDR Stack APIs. Bluetooth BR/EDR Stack APIs can be divided into several modules: Device Management APIs, Service Discovery Protocol (SDP) APIs and BR/EDR Link Management APIs. Applications can interact with BR/EDR stack through Device Management/SDP/Link Manager APIs & Callback message.

4.8 Application Samples

4.8.1 Audio Demo Application Note

The purpose of this document is to give an overview of the audio demo application. The audio demo project gives a simple example of how to use VP, ringtone, and line-in. This project demonstrates how to play VP and Ringtone, and how to start or stop Line-in.

4.8.2 BLE Sample Project Application Note

The purpose of this document is to give an overview of the BLE sample application. There are four GAP roles (broadcaster, observer, peripheral, central) defined for devices operating over an LE physical transport. SDK provides a corresponding demo application for the user's reference in development.

4.8.3 BT Audio Application Note

The purpose of this document is to give an overview of the BT audio demo application. The BT audio project gives a simple example of how to use A2DP and HFP. This project implements Source (SRC) and Sink (SNK) roles for A2DP. And for HFP, Audio Gateway (AG) and Hands-Free unit (HF) are implemented.

4.8.4 BT Audio Transceiver Application Note

BT Audio Transceiver(BT_Audio_Trx) is an application developed based on Common SDK, it has the capability of BT, BLE, and Audio. The host (external MCU) can communicate with BT_Audio_Trx through the UART interface.

4.8.5 BT Audio Transceiver ACI User Manual

This Part of the specification describes the functional specifications for the Application Controller interface (ACI). The ACI provides a uniform command method for the Host to access Device capabilities and control connections to other Devices. The commands and events are sent between the Host and the Device.

4.8.6 BT Audio Transceiver Design Spec

This article focuses on the design spec of bt_audio_trx. It is written to help users easily and completely understand the code flow of the application.

4.8.7 DTM User Manual

The purpose of this document is to give an overview of Direct Test Mode. Direct Test Mode is used to test the RF PHY layer of Bluetooth low-energy devices. DTM controls the Device Under Test and provides a report back to the Tester. DTM shall be set up by one of the two alternate methods: over HCI or through a 2-wireUART interface. Each DUT shall implement one of the two DTM methods.

4.8.8 BT HID Demo Application Note

The purpose of this document is to give an overview of the HID Demo application. The HID Demo project gives a simple example of how to use HID. This project implements HID mouse device role.

4.8.9 BT MAP Demo Application Note

The purpose of this document is to give an overview of the BT MAP Demo application. The BT MAP Demo project gives a simple example on how to use MAP. This project implements MAP MCE role.

4.8.10 PBAP Demo Application Note

The purpose of this document is to give an overview of the PBAP Demo application. The PBAP Demo project gives a simple example on how to use PBAP. This project implements client role for PBAP.

4.8.11 iAP Demo Application Note

The purpose of this document is to give an overview of the iAP Demo application. The iAP Demo project gives a simple example on how to use iAP. This project implements accessory role for iAP.

4.8.12 LE Audio Sample Project Application Note

The purpose of this document is to give an overview of the LE Audio Demo application. CAP defines the roles of Acceptor, Initiator, and Commander.

The Acceptor application implements a very simple LEA CAP Acceptor device and it can be used as a framework to develop many different CAP acceptor-role based applications.

The Commander application implements a very simple LEA CAP Commander device and it can be used as a framework to develop many different CAP commander-role based applications.

The Initiator application implements a very simple LEA CAP Initiator device and it can be used as a framework to develop many different CAP initiator-role based applications.

4.8.13 OTA Application Note

OTA (Over The Air) represents the technology that applies Bluetooth to update images (code and data) that runs in RTL87X3E Flash. RTL87X3E supports two OTA modes for Android and IOS: OTA via BLE(IOS) and OTA via SPP(Android).

4.8.14 SPP Demo Application Note

The purpose of this document is to give an overview of the Serial Port Profile(SPP) and demo application. The SPP demo project implements a very simple SPP application and can be used as a framework for developing many different SPP applications.

4.8.15 Platform Power Test User Manual

This article introduces how to measure the platform power consumption, including supported test scenarios, the settings for RCFG and EVB, and how to use CMD to measure power consumption. Users can refer to platform power consumption to know about how much power some functions generate, such as BLE/BT test scenarios. In addition, when users have issues with power consumption, they can also check whether there are problems with hardware or APP Settings through platform power consumption, such as static power test scenarios.

4.8.16 USB Audio Demo Application Note

The purpose of this document is to give an overview of the USB audio application. The USB audio project implements a very simple USB audio device.

4.8.17 USB HID Demo Application Note

The purpose of this document is to give an overview of the USB HID application. The USB HID project implements a very simple USB HID device.

4.9 Tool

4.9.1 Development Tool

4.9.1.1 MCU Config Tool User Guide

This article explains the functions, usage, and settings of the MCU Config Tool for Realtek Bluetooth Audio Chip. Configurable BT settings and peripheral control are offered by REALTEK Bluetooth MCU. By using MCU Config Tool during the development stage, the user can easily configure many MCU parameters.

MCU Config Tool divides the setting elements into various tabs, such as HW Feature, Audio Route, General, System Configuration, Charger, Ringtone, RF TX, and so forth. These configurations will be described in the following sections.

4.9.1.2 DSP Config Tool User Guide

This documentation provides detailed guidance on the graphical user interface(GUI) to fine-tune the behavior related to the DSP FW part in the RTL87x3E IC series. This GUI tool, the DSP configuration tool, allows developers to configure parameters of built-in signal processing algorithms for voice and audio applications as well as the hardware peripheral interfaces connected to the DSP core, including an analog-to-digital converter (ADC), digital-to-analog converter (DAC) and I2S.

In addition, this DSP configuration tool also provides user-friendly parameter fine-tuning functions, which can greatly shorten the turnaround time of parameter-tuning.

Finally, this tool also supports the generation of DSP configuration files for the RTL87x3E toolchain including flash download and mass-production software tools.

4.9.1.3 MPPG Tool User Guide

MP PG Tool supports flash programming for RTL87XX series chips. There are two modes of downloading images to the chip. One is MP Mode, which is for factory mass production; while the other is RD Mode, which is for factory R&D to do some debugs about their image downloading to chip.

4.9.1.4 Debug Analyzer User Guide

DebugAnalyzer is a tool used for capturing and decoding logs of the Realtek Bluetooth SOC chip. This document provides detailed guidance on how to set it up and use it.

4.9.1.5 BT Audio Transceiver AciHostCLI Tool

AciHostCLI is a test tool running on PC, which is used to demo ACI usage. It can perform cmd/event communication test with ACI Device through USB serial port.

4.9.2 RF Test and Certification Tool and MP Tool

4.9.2.1 RF Test and Certification Tool

This document introduces how to set up the testing environment for the RTL87x3E RF, including the test mode specified in the BT specification and the single-tone test mode. It can be used for the whole series of RTL87x3E chips. FCC/BQB/CE/SRRC/KC and other certifications can use this tool to assist in testing. Contact Realtek Bluetooth FAE if any problem arises in the use of the test.

4.9.2.2 MP Tool

This document introduces and provides Realtek's mass production solution for the RTL87x3E MP, including the MP flow and test item in this MP solution. It can be used for the whole series of RTL87x3E chips. Contact Realtek Bluetooth FAE if any problem arises in the use of the test.
