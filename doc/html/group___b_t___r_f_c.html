◆ bt_rfc_conn_req()

bool bt_rfc_conn_req ( uint8_t  bd_addr[6],
uint8_t  local_server_chann,
uint8_t  remote_server_chann,
uint16_t  frame_size,
uint8_t  init_credits 
)

Send a request to create a RFCOMM connection.

Parameters
[in]bd_addrRemote BT address.
[in]local_server_channThe local server channel of the service that sending this request.
[in]remote_server_channThe remote server channel which can be found from the SDP info.
[in]frame_sizeThe maximum frame_size supported by local device.
[in]init_creditsThe number of packet that remote can be send. This parameter is used for flow control. A sending entity may send as many frames on a RFCOMM channel as it has credits; if the credit count reaches zero, the sender will stop and wait for further credits from peer.
Returns
The status of sending connection request.
Return values
trueRequest has been sent successfully.
falseRequest was fail to send.

◆ bt_rfc_conn_cfm()

bool bt_rfc_conn_cfm ( uint8_t  bd_addr[6],
uint8_t  local_server_chann,
bool  accept,
uint16_t  frame_size,
uint8_t  init_credits 
)

Send a confirmation to accept or reject the received RFCOMM connection request.

Parameters
[in]bd_addrRemote BT address.
[in]local_server_channLocal server channel.
[in]acceptConfirmation message.
  • true Accept the received RFCOMM connection request.
  • false Reject the received RFCOMM connection request.
[in]frame_sizeThe maximum frame_size supported by local device.
[in]creditsThe number of packet that remote can be send. This parameter is used for flow control. A sending entity may send as many frames on a RFCOMM channel as it has credits; if the credit count reaches zero, the sender will stop and wait for further credits from peer.
Returns
The result of sending confirmation.
Return values
trueThe confirmation has been sent successfully.
falseThe confirmation was fail to send.
