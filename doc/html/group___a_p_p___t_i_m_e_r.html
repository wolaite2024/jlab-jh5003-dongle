Detailed Description

Register timeout callback functions, start and stop software timers.

The APP Timer Module allows to register timeout callback functions which will be called when timeout, also provides start/stop timer functions. APP Timer Module provides a method for APP to handle timeout events in its own task.

Register a timeout callback function in APP Timer Module. A module ID is returned which will be used when start timer by calling app_start_timer. When timeout event occurs, APP Timer Module will find the exact timeout function by module ID and execute it.

Create a new APP timer and start it. This allocates the storage required by the new timer, starts this timer if create successfully, and returns a handle by which the new timer can be referenced. APP timer is always a one-shot timer

app_timer.h

Parameters
[out]p_idxUsed to pass back an index by which the created timer can be referenced.
[in]p_nameA descriptive name for the timer.
[in]module_idTimer module ID assigned by APP Timer Module when register timer callback function using app_reg_timer_cb. module ID would be used to identify which timeout function will be execute.
[in]timer_evtTimer event assigned by module which starts this timer.
[in]paramParamer assigned by module which start this timer. Typically timer event and paramter would be used in the timer callback function to identify which timer expired when the same callback function is assigned to more than one timer.
[in]reloadUsed to set the timer as a periodic or one-shot timer.
[in]timeout_msTimeout value in milliseconds.
Returns
Operation result error.h.
Return values
0Operation success.
OthersOperation failure.
