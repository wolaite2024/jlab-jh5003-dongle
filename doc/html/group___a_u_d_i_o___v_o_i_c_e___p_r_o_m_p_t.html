Detailed Description

Play and control the Voice Prompt.

Voice Prompt (VP) plays voice files with the required languages. It is used when prompting for notification, alarm, user-confirmation, as well as other situations.

Typedef Documentation

◆ T_VOICE_PROMPT_MODE

typedef enum t_voice_prompt_mode T_VOICE_PROMPT_MODE

Define the Voice Prompt modes.

Voice Prompt has three modes which are audible mode, silent mode, and volume fixed mode. Audible mode is Voice Prompt's default mode that Voice Prompt can be played and stopped, and Voice Prompt volume can be adjusted, muted and unmuted. Silent mode will disable Voice Prompt and mute Voice Prompt volume that Voice Prompt cannot be played and its volume cannot be adjusted. Fixed mode will make Voice Prompt volume unadjustable.

◆ voice_prompt_play()

bool voice_prompt_play ( uint8_t  index,
T_VOICE_PROMPT_LANGUAGE_ID  language,
bool  relay,
uint32_t  addr,
uint32_t  len 
)

Play a Voice Prompt.

The Voice Prompt is first pushed into the notification queue inside Audio Subsystem. When there are no pending voice prompts or ringtones in the notification queue, this Voice Prompt will be played immediately, otherwise, it will be scheduled until all the preceding voice prompts or ringtones are popped from the notification queue. Upper layer will be notified by the audio event AUDIO_EVENT_VOICE_PROMPT_STARTED when started, and by the audio event AUDIO_EVENT_VOICE_PROMPT_STOPPED when stopped.

Note
The notification queue inside Audio Subsystem is basically a first-in, fist-out (FIFO) queue, So all the voice prompts and ringtones pending in the notification queue will be played in sequence. Upper layer can use voice_prompt_cancel to cancel these voice prompts with the required index, or voice_prompt_flush to flush all the voice prompts pending in the notification queue. This action will be permitted only when the Voice Prompt mode is VOICE_PROMPT_MODE_AUDIBLE or VOICE_PROMPT_MODE_FIXED. See T_VOICE_PROMPT_MODE for details.
Parameters
[in]indexThe type of the playing Voice Prompt.
[in]languageLanguage for the playing Voice Prompt T_VOICE_PROMPT_LANGUAGE_ID.
[in]relayRelay the Voice Prompt to the peer identical device.
Returns
The result of playing Voice Prompt.
Return values
trueThe Voice Prompt was played successfully.
falseThe Voice Prompt was failed to play.

Cancel the specific Voice Prompt in the queue.

All the voice prompts with the required index pending in the notification queue inside Audio Subsystem will be canceled. Note that if there is a playing Voice Prompt with the same index, this playing Voice Prompt will also be stopped immediately.

Note
This action will be permitted only when the Voice Prompt mode is VOICE_PROMPT_MODE_AUDIBLE or VOICE_PROMPT_MODE_FIXED. See T_VOICE_PROMPT_MODE for details.

trueThe Voice Prompt was canceled successfully.
falseThe Voice Prompt was failed to cancel.

◆ voice_prompt_flush()

bool voice_prompt_flush ( bool  relay)

Flush all the pending voice prompts in the queue.

All the voice prompts pending in the notification queue inside Audio Subsystem will be flushed immediately. If there is a Voice Prompt currently under playing, it will not be affected by this flush operation.

Note
This action will be permitted only when the Voice Prompt mode is VOICE_PROMPT_MODE_AUDIBLE or VOICE_PROMPT_MODE_FIXED. See T_VOICE_PROMPT_MODE for details.
Parameters
[in]relayRelay the Voice Prompt flushing action to the peer identical device.
Returns
The result of flushing the voice prompts.
Return values
trueThe Voice Prompt was flushed successfully.
falseThe Voice Prompt was failed to flush.
