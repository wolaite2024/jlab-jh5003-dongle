bool aes128_cbc_encrypt ( uint8_t *  plaintext,
const uint8_t  key[16],
uint8_t *  encrypted,
uint32_t *  p_iv,
uint32_t  data_word_len 
)

Encrypt the speicified plaintext by AES CBC mode with a 128-bit key.

Parameters
plaintextSpecify the plaintext to be encypted.
keySpecify the 128-bit key to encrypt the plaintext.
encryptedSpecify the output buffer to store the encrypted data.
p_ivSpecify the initialization vector (IV) for AES CBC mode.
data_word_lenSpecify the word length of the data. The data length must be a multiple of 4.
Returns
Encryption result.
Return values
trueSuccess.
falseFail.
Note
The least significant octet of encrypted data corresponds to encypted[0].

Decrypt data by AES CBC DMA mode with 128-bit key.

Experimental Added Since 2.13.0.0:
Parameters
inputSpecify the source address of the data to be decypted.
outputSpecify the output buffer to store the plain data.
data_word_lenSpecify the word length of the data. The data length must be a multiple of 4.
keySpecify the 128-bit key to decrypt the data.
p_ivSpecify the initialization vector (IV) for AES CBC mode.
cbackRegister the callback function when the total AES DMA operation has been finished.
Returns
The AES CBC decryption result by DMA mode.
Return values
trueSuccess.
falseFail.
Note
The RX DMA channel and TX DMA channel are requested and used by HW AES DMA operation. The RX DMA channel and TX DMA channel should be released in the HW AES DMA done callback.
